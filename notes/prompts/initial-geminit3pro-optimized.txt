**Role:** Senior Graphics Architect & Project Manager
**Goal:** Design a production-grade, high-performance, cross-platform hybrid
raytracing render graph library for desktop games.
**Output Format:** Structured Markdown documentation tree with Mermaid diagrams. **NO CODE IMPLEMENTATION.** Scaffolding/Interfaces only.

### 1. Project Constraints & Scope
* **Platforms:** MacOS (Initial Target via Metal), Windows, SteamOS. (No Console/Mobile).
* **Hardware:** Modern GPU focus. Bindless resources, Mesh Shaders, ray tracing, etc.
* **Excluded:** Legacy pipelines (Geometry shaders, Tessellation), Compatibility layers (MoltenVK).
* **Tech Stack:**
    * **Frontend:** 100% Safe Rust (Declarative API).
    * **Backend:** Unsafe C++ (High perf, platform-specific) via `cxx.rs`.
      Requires Direct3D 12, Vulkan 1.4 with extensions, or Metal 4 at minimum.
    * **Shaders:** Naga (Platform-neutral AST) for generating HLSL/Metal.
    * **Wrappers:** Official C++ wrappers (e.g., `metal-hpp`, `vulkan-hpp`).

### 2. Architecture Specifications
* **Thread Model:**
    * **User Thread:** 100% Safe, high-level debugging.
    * **Background Threads:** Unsafe Render & IO threads.
    * **Synchronization:** Careful management of shared assets between queues (e.g., split-screen).
* **Render Graph Lifecycle:**
    1.  **Build Phase:** Declarative, frame-invariant description.
    2.  **Compile Phase:** Generate "Execution Plan" (Optimized Index).
        * *Culling:* Remove unsupported features, unused nodes (transitive), and performance-heavy nodes based on budget.
    3.  **Execution Phase:** Frame-specific updates, Async command encoding, Resource loading.
* **IO & Streaming:**
    * Pool of background workers using Transfer Queues.
    * APIs: `MTLIOCommandBuffer`, DirectStorage.
    * **Streaming:** Tiles for massive terrains, 3D textures for voxels.
    * **Async Compute:** Integrated with IO for real-time asset generation (Mesh shaders) and readback.
* **Data Handling:**
    * Dynamic CPU-side distance sorting (generic, e.g., for transparency).
    * Serializable objects for Shader Graph authoring (Visual tool driven, no user code).

### 3. Feature Requirements (Exhaustive)
* **Core:** Direct lighting, Frustrum/Backface/Occlusion culling, Orthographic/Perspective projections, Instancing, Render-to-texture, Cubemaps.
* **Lighting/PBR:** Forward+, Deferred, PBR (Multi-map/BSDF), Real-time GI (Irradiance caching), Ray tracing (Hardware), Ray traced reflections.
* **Shadows:** Cascaded Shadow Maps. Soft shadows. Ray traced indirect lighting.
* **Volumetrics/Environment:** Ray-marched volumetrics, Procedural Sky (Sun, Moon, Stars, Clouds, Fog), God rays, Rayleigh scattering, Water (Deformation/Sim).
* **Geometry:** Meshlet-based virtualized geometry, Procedural assets (Real-time), Spline paths, Infinite streamable universes, Planetary-scale voxel worlds, Detailed open-world terrains.
* **Effects:** Ambient Occlusion, Subsurface scattering, Transparent objects,
  Alpha mask cutouts.
* **Animations:** Weighted, instanced skeletal animation, morph shapes, clips,
  animation state machines, procedural animations, custom animation curves
* **UI/2D:** Vector/Bitmap-based retained UI, 2D/2.5D support.
* **Tooling:** Resource CPU readback (persistence), Data-driven shader-graph
  authoring. Advanced Blender3D compatibility.

### 4. Planning & Testing Directives
* **Subagent Simulation:** Spawn one subagent per feature item to investigate, plan, and list requirements.
* **Testing Strategy:**
    * **TDD:** Unit tests for all modules.
    * **End-to-End:** Real GPU usage, render to buffer/video, validate outputs
    * **Validation:** Snapshot comparison (Frame-by-frame diffs). Minor deviation tolerated
    * **Fuzzing:** Test failure scenarios/improper inputs on real GPU.
* **Instrumentation:** Allocation tracking, Resource tracking, GPU counters, CPU
  counters, graph execution tracing, Shader logging/debugging.
