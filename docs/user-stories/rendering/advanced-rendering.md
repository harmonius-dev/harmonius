# User Stories -- 2.5 Advanced Rendering (Ray Tracing and GI)

## Stories

| ID | Persona | Story | Acceptance Criteria | Features | Requirements |
|----|---------|-------|---------------------|----------|--------------|
| US-2.5.1.1 | engine developer (P-26) | I want to build BLAS from meshlet geometry with post-build compaction and rebuild TLAS each frame for dynamic objects | RT features have an up-to-date acceleration structure with minimal VRAM waste |  |  |
| US-2.5.1.2 | engine tester (P-27) | I want to verify that BLAS construction respects per-platform memory budgets (limited on Apple A17+/M3+ and Switch 2 Ampere) | RT scenes do not exceed mobile or Switch VRAM allocations |  |  |
| US-2.5.2.1 | player (P-23) | I want hybrid reflections that combine screen-space rays with hardware RT for off-screen content | mirrors and polished floors show correct reflections even when the reflected object is not on screen |  |  |
| US-2.5.2.2 | engine developer (P-26) | I want to measure the cost of RT reflection rays at varying roughness thresholds | I can tune the SSR-to-RT handoff point to balance quality and GPU budget per platform |  |  |
| US-2.5.2.3 | engine tester (P-27) | I want to disable both SSR and RT and confirm that reflection probes provide correct fallback reflections on all surfaces | mobile devices without SSR still show plausible reflections |  |  |
| US-2.5.3.1 | player (P-23) | I want one-bounce indirect diffuse lighting that bleeds wall color onto adjacent surfaces | environments feel naturally lit with warm and cool tones rather than flat ambient |  |  |
| US-2.5.3.2 | engine developer (P-26) | I want to verify that 0.25 spp RT indirect lighting accumulates over 8-16 frames to produce stable noise-free GI | indirect lighting converges quickly enough for real-time use |  |  |
| US-2.5.4.1 | environment artist (P-8) | I want DDGI probes that recast rays and update irradiance when I reposition lights in the editor | indirect lighting responds to light changes in real time without rebaking |  |  |
| US-2.5.4.2 | engine tester (P-27) | I want to verify that desktop uses 8m probe spacing with 64 rays/probe and high-end uses 4m/128+ rays | GI quality scales correctly with hardware capability |  |  |
| US-2.5.5.1 | technical artist (P-13) | I want a progressive path tracer that renders offline- quality images with full light transport and all material types | I can compare real-time rendering against a reference to identify approximation artifacts |  |  |
| US-2.5.5.2 | engine tester (P-27) | I want to render a white-furnace test scene with the path tracer and verify that energy is conserved (no darkening or brightening over bounces) | the reference renderer produces physically accurate results |  |  |
| US-2.5.6.1 | player (P-23) | I want ray-traced subsurface transmission through thin body parts like ears, fingers, and nostrils when backlit | characters look lifelike in strong directional lighting |  |  |
| US-2.5.6.2 | engine tester (P-27) | I want to disable RT and confirm that subsurface scattering falls back to preintegrated LUT on mobile and Switch without visual artifacts | skin rendering degrades gracefully |  |  |
| US-2.5.7.1 | environment artist (P-8) | I want surfel-based GI that discretizes scene geometry from the G-buffer each frame without precomputation | fully dynamic environments with destructible objects receive correct indirect lighting |  |  |
| US-2.5.7.2 | engine developer (P-26) | I want to measure per-frame GPU cost of surfel GI at 2 clipmap levels (desktop) versus 4+ levels (high-end) | I can validate that multi-scale surfel density meets performance targets per tier |  |  |
| US-2.5.8.1 | environment artist (P-8) | I want ReSTIR DI to sample thousands of lights with physically correct soft shadows from every source | I can author dense light environments (concert halls, cityscapes) without per-light shadow cost |  |  |
| US-2.5.8.2 | engine developer (P-26) | I want to measure per-pixel reservoir memory footprint during ReSTIR DI+GI and confirm it fits within a 2 GB render target budget on desktop | ReSTIR operates within VRAM constraints |  |  |
| US-2.5.9.1 | player (P-23) | I want multi-bounce real-time path tracing with neural denoising for reflections, AO, GI, and refractions | the game looks as close to cinematic quality as possible during live gameplay |  |  |
| US-2.5.9.2 | engine tester (P-27) | I want to confirm that the path tracer falls back to rasterized GI after the final bounce rather than terminating to black | path-traced scenes maintain ambient fill in shadowed areas |  |  |
| US-2.5.10.1 | engine developer (P-26) | I want opacity micromaps to annotate per-micro-triangle opacity for vegetation and hair | hardware skips any-hit shader invocations for classified triangles and RT performance improves by 20-40% |  |  |
| US-2.5.10.2 | engine tester (P-27) | I want to run on Ada Lovelace and RDNA3+ GPUs and confirm that opacity micromaps are enabled automatically | supported hardware gets the RT performance benefit without manual configuration |  |  |
| US-2.5.11.1 | engine developer (P-26) | I want shader execution reordering to group similar RT shader invocations into coherent wavefronts | divergent material shaders after ray intersection run with improved GPU occupancy |  |  |
| US-2.5.11.2 | engine tester (P-27) | I want to verify that SER activates on Ada Lovelace+ via VK_NV_ray_tracing_invocation_reorder and is skipped on unsupported hardware | the feature is conditionally enabled without driver errors |  |  |
| US-2.5.12.1 | technical artist (P-13) | I want a neural denoiser to replace hand-tuned spatiotemporal filters for RT and path-traced output | noisy 1-spp rays produce temporally stable, artifact-free images |  |  |
| US-2.5.12.2 | engine tester (P-27) | I want to run on a GPU without Tensor Cores and confirm that the engine falls back to NRD bilateral/wavelet denoising | RT denoising works on all desktop GPUs regardless of neural hardware |  |  |
| US-2.5.13.1 | player (P-23) | I want ray-traced lens flare that simulates real lens element refraction to produce ghost and halo artifacts | looking toward the sun or bright explosion produces convincing optical effects |  |  |
| US-2.5.13.2 | engine tester (P-27) | I want to disable RT and verify that lens flare falls back to the image-based approximation (F-2.9.8) | mobile and Switch still show lens flare effects |  |  |
| US-2.5.14.1 | environment artist (P-8) | I want voxel-based GI that works without RT hardware by voxelizing the scene and relighting the voxel cache for time-of-day changes | Switch and older desktop GPUs still get convincing indirect lighting |  |  |
| US-2.5.14.2 | engine developer (P-26) | I want to measure 3D texture VRAM for voxel GI at 64^3 (Switch), 128^3 (desktop), and 256^3 (high-end) grid resolutions | I can verify each tier fits within its platform VRAM budget |  |  |
| US-2.5.15.1 | engine developer (P-26) | I want a runtime-trained neural network that predicts remaining light transport after 2-3 bounces | path-traced output has less noise with fewer full-length paths required |  |  |
| US-2.5.15.2 | engine tester (P-27) | I want to confirm that the neural radiance cache requires Tensor Core hardware and 12+ GB VRAM, and is disabled by default on all other configurations | the feature does not activate on unsupported hardware |  |  |
| US-2.5.16.1 | player (P-23) | I want stochastic screen-space reflections importance-sampled from the microfacet BRDF | rough metal surfaces show elongated, physically correct specular reflections rather than sharp mirror images |  |  |
| US-2.5.16.2 | engine tester (P-27) | I want to compare stochastic SSR at quarter-res (Switch), half-res (desktop), and half-res with spatial+temporal denoiser (high-end) | each platform tier produces acceptable reflection quality |  |  |
