# User Stories -- 15.15 Engine Launcher

## Stories

| ID           | Persona                    |
|--------------|----------------------------|
| US-15.15.1.1 | game designer (P-5)        |
| US-15.15.1.2 | build engineer (P-16)      |
| US-15.15.2.1 | engine developer (P-26)    |
| US-15.15.2.2 | game designer (P-5)        |
| US-15.15.3.1 | game designer (P-5)        |
| US-15.15.3.2 | level designer (P-6)       |
| US-15.15.4.1 | engine developer (P-26)    |
| US-15.15.4.2 | game designer (P-5)        |
| US-15.15.5.1 | game designer (P-5)        |
| US-15.15.5.2 | build engineer (P-16)      |
| US-15.15.6.1 | build engineer (P-16)      |
| US-15.15.6.2 | game designer (P-5)        |

1. **US-15.15.1.1** — **As a** game designer (P-5), **I want** a standalone launcher to install,
   update, and manage multiple engine versions side by side, **so that** projects pin to a specific
   version.

2. **US-15.15.1.2** — **As a** build engineer (P-16), **I want** per-version disk space display with
   one-click uninstall, **so that** I manage storage proactively.

3. **US-15.15.2.1** — **As a** engine developer (P-26), **I want** automatic project upgrade
   pipelines running versioned migration scripts, **so that** projects update safely.

4. **US-15.15.2.2** — **As a** game designer (P-5), **I want** a pre-upgrade backup and migration
   report, **so that** I can review and revert if needed.

5. **US-15.15.3.1** — **As a** game designer (P-5), **I want** a project browser with thumbnails and
   a creation wizard with genre templates, **so that** new projects start quickly.

6. **US-15.15.3.2** — **As a** level designer (P-6), **I want** genre templates to pre-configure
   gameplay systems and starter content, **so that** I begin with a playable foundation.

7. **US-15.15.4.1** — **As a** engine developer (P-26), **I want** a TOML project file at the
   project root registering as the default handler for double-click open, **so that** projects
   launch in the correct engine version.

8. **US-15.15.4.2** — **As a** game designer (P-5), **I want** the project file to be human-readable
   and version-controlled, **so that** changes are tracked with the project.

9. **US-15.15.5.1** — **As a** game designer (P-5), **I want** global preferences synced across
   machines via the cloud service, **so that** my theme and keybindings follow me.

10. **US-15.15.5.2** — **As a** build engineer (P-16), **I want** multi-account support for
    switching between personal and organization accounts, **so that** credentials are separate.

11. **US-15.15.6.1** — **As a** build engineer (P-16), **I want** a collaboration setup wizard
    connecting Git, LFS, CRDT server, and shared cache, **so that** team infrastructure configures
    in one flow.

12. **US-15.15.6.2** — **As a** game designer (P-5), **I want** new team members to inherit setup
    from the project file, **so that** onboarding requires no manual configuration.
