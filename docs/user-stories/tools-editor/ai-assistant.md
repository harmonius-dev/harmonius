# User Stories: AI Editor Assistant

## US-15.9.1 Voice-Controlled Scene Editing

**As a** designer, **I want** to issue voice commands like "place a directional light
facing east at 45 degrees" and have the assistant create the entity with correct
orientation, **so that** I can manipulate scenes hands-free without navigating menus.

## US-15.9.2 Speech-to-Text for Voice Commands

**As a** designer, **I want** my voice to be captured by the microphone and transcribed
accurately in real time, **so that** I can issue voice commands to the editor.

## US-15.9.3 Natural Language Command Interpretation

**As a** designer, **I want** the assistant to understand natural language commands with
conversational context (e.g., "rotate it 10 degrees north" references the last object),
**so that** I can interact naturally without memorizing rigid command syntax.

## US-15.9.4 Choose Voice Activation Mode

**As a** designer, **I want** to choose between push-to-talk and always-listening voice
activation modes in my preferences, **so that** I can use voice commands in a way that
suits my work environment.

## US-15.9.5 Agent-Driven Automation via Headless API

**As a** DevOps engineer, **I want** AI agents to drive the editor through a headless API
without a visible display, **so that** I can automate content generation and testing.

## US-15.9.6 Run Multiple Agents Concurrently

**As a** DevOps engineer, **I want** multiple AI agents to operate concurrently in isolated
contexts, **so that** parallel automation tasks do not interfere with each other.

## US-15.9.7 Integrate Agents Into CI/CD Pipelines

**As a** DevOps engineer, **I want** headless agents to integrate with CI/CD pipelines for
automated content generation and regression testing, **so that** repetitive asset production
runs unattended as part of the build process.
