# User Stories -- 6.5 VR and Spatial Input

## US-6.5.1 Look Around in VR Without Sickness
**As a** player, **I want** 6DOF head tracking at up to 120 Hz with late-latching pose
updates and tracking loss detection, **so that** I experience low motion-to-photon latency
and never get disoriented by a tracking failure.

## US-6.5.2 Use VR Controllers Like Gamepad
**As an** input designer, **I want** VR motion controller inputs mapped through the same
action system as gamepads, **so that** shared input configurations work across flat-screen
and VR modes without per-mode authoring.

## US-6.5.3 Interact with Hands in VR
**As a** player, **I want** camera-based hand tracking with a full skeletal model and
predefined gesture recognition (pinch, grab, point), **so that** I can interact naturally
in VR without physical controllers when supported by the headset.

## US-6.5.4 Select UI Elements by Looking
**As a** player, **I want** eye tracking providing gaze direction for look-and-select UI
interaction and foveated rendering, **so that** I can navigate menus by looking and the GPU
allocates detail where I am actually looking.

## US-6.5.5 Feel VR Interactions in Each Hand
**As a** player, **I want** per-hand haptic feedback with amplitude, frequency, and spatial
proximity-based intensity, **so that** sword impacts feel different in my right hand from
shield blocks in my left, and vibration increases as I reach toward objects.
