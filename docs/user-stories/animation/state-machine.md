# User Stories — 9.4 Animation State Machine

## US-9.4.1 Author Complex Character Behaviors in a Visual Graph
**As a** designer, **I want** a declarative state graph with blend spaces, montages, sync
groups, and AI integration authored entirely in the visual editor, **so that** I can build
responsive character animation without writing code.

| Acceptance criteria | Features | Requirements |
|---|---|---|
| CPU-side state graph with GPU-evaluated blending | F-9.4.1 | R-9.4.1 |
| Sync-marker-aligned transitions with blend profiles | F-9.4.2 | R-9.4.2 |
| Reusable sub-state machines with entry/exit points | F-9.4.3 | R-9.4.3 |
| Parallel layers with per-bone masks and blend modes | F-9.4.4 | R-9.4.4 |
| Named parameters drive transitions via expressions | F-9.4.5 | R-9.4.5 |
| Sync groups keep walk/run cycles phase-aligned | F-9.4.6 | R-9.4.6 |
| Montages override state machine on bone groups | F-9.4.7 | R-9.4.7 |
| 1D and 2D blend spaces with barycentric weights | F-9.4.8 | R-9.4.8 |
| Aim offsets with IK integration for weapon alignment | F-9.4.9 | R-9.4.9 |
| AI behavior trees query and trigger animation states | F-9.4.10 | R-9.4.10 |
| All authoring via visual editor (no-code) | F-9.4.1 | R-X.9.5 |
