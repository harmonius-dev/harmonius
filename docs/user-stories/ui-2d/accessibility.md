# User Stories — 10.6 Accessibility

| ID         | Persona                 |
|------------|-------------------------|
| US-10.6.1  | player (P-23)           |
| US-10.6.2  | player (P-23)           |
| US-10.6.3  | player (P-23)           |
| US-10.6.4  | game designer (P-5)     |
| US-10.6.5  | player (P-23)           |
| US-10.6.6  | player (P-23)           |
| US-10.6.7  | player (P-23)           |
| US-10.6.8  | technical artist (P-13) |
| US-10.6.9  | player (P-23)           |
| US-10.6.10 | game developer (P-15)   |

1. **US-10.6.1** — **As a** player (P-23), **I want** the widget tree exposed to screen readers so
   they announce labels, roles, and states, **so that** I navigate the game with assistive
   technology.
2. **US-10.6.2** — **As a** player (P-23), **I want** configurable subtitles with speaker
   identification and closed captions with directional indicators, **so that** I understand all
   audio visually.
3. **US-10.6.3** — **As a** player (P-23), **I want** colorblind modes for protanopia, deuteranopia,
   and tritanopia with alternative visual cues, **so that** I perceive color-coded information.
4. **US-10.6.4** — **As a** game designer (P-5), **I want** alternative visual cues (patterns,
   shapes, icons) for each color-coded gameplay element, **so that** information does not rely
   solely on color.
5. **US-10.6.5** — **As a** player (P-23), **I want** to scale the entire UI from 80% to 250% with
   proportional scaling and high-contrast mode, **so that** I read and interact comfortably.
6. **US-10.6.6** — **As a** player (P-23), **I want** complete remapping of all bindings including
   one-handed layouts and switch-device scanning, **so that** I play with my available capabilities.
7. **US-10.6.7** — **As a** player (P-23), **I want** incoming chat and notifications converted to
   speech with per-channel controls, **so that** I follow conversations without reading.
8. **US-10.6.8** — **As a** technical artist (P-13), **I want** to define accessible default styles
   for every widget that meet WCAG contrast ratios, **so that** accessibility is built into the
   design.
9. **US-10.6.9** — **As a** player (P-23), **I want** hold-to-toggle conversion for sustained key
   presses, **so that** I activate abilities without holding buttons.
10. **US-10.6.10** — **As a** game developer (P-15), **I want** WCAG 2.1 AA compliance for all menu
    UI with AAA for critical gameplay information, **so that** the game is accessible to the widest
    audience.
