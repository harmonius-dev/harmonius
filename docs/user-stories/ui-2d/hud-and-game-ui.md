|---|---|---|---|---|---|

| US-10.3.6 | Player (P-23) | I want configurable action bar grids with radial cooldown sweeps, charge counts, range/usability tinting, and keybind labels, so that I know exactly when abilities are ready without frame-inaccurate visual delays. | Radial cooldown sweeps frame-accurate with simulation; charge counts and range/usability tinting; multiple action bar pages and stance-conditional bars; keybind labels displayed per slot | F-10.3.3 | R-10.3.3 |

## Map System

| ID | Persona | Story | Acceptance Criteria | Features | Requirements |
|---|---|---|---|---|---|
| US-10.3.24 | Developer (P-15) | I want the minimap to auto-generate from terrain, biome, building footprint, and road spline data without artist intervention, so that procedurally generated worlds have functional minimaps immediately. | Minimap colorized by biome/material type; building footprints from collision geometry; roads from road spline data; newly streamed chunks extend minimap automatically | F-10.3.12 | R-10.3.12 |
| US-10.3.25 | Player (P-23) | I want the minimap to reveal terrain progressively as I explore with fog of war hiding undiscovered areas, so that exploration feels rewarding and the map serves as a record of where I have been. | Fog of war hides unexplored minimap areas; explored areas remain visible after leaving; voxel worlds show slice at player elevation; minimap texture scales per platform (256 mobile, 512 desktop) | F-10.3.12 | R-10.3.12 |
| US-10.3.26 | Player (P-23) | I want a world map with terrain, biome, political, road, settlement, water, and fog-of-war layers that I can zoom, pan, and search, so that I can plan routes, find locations, and set waypoints. | All 7 map layers rendered (terrain through annotations); continuous zoom from world overview to local detail; click-to-set-waypoint and click-to-fast-travel; search-by-name for discovered locations | F-10.3.13 | R-10.3.13 |
| US-10.3.27 | Player (P-23) | I want the world map to render on a zoomable globe with zoom-to-surface transitions for planet-scale worlds, so that the map matches the scale and curvature of the game world. | Globe rendering for planet-scale worlds; smooth zoom-to-surface transitions; tiled image pyramids for efficient zoom rendering; globe rendering desktop/console only | F-10.3.13 | R-10.3.13 |
| US-10.3.28 | Engine developer (P-26) | I want to implement a world map generation pipeline that produces tiled image pyramids from terrain, biome, political, and infrastructure data, so that maps render efficiently at all zoom levels with cached tiles. | Map generated from procedural generation output; tiled image pyramids cached for zoom rendering; mobile uses lower-resolution tiles and fewer zoom levels | F-10.3.13 | R-10.3.13 |
| US-10.3.29 | Artist (P-8) | I want to apply post-processing styles to the world map including parchment, painted, tactical, satellite, and schematic modes, so that the map reinforces the game's visual identity. | At least 5 post-processing styles available; parchment style with sepia tone and ink-drawn lines; artistic layers composed in the visual editor | F-10.3.14 | R-10.3.14 |
| US-10.3.30 | Artist (P-8) | I want to substitute static hand-painted map images for specific zones while dynamic data layers (player position, quest markers, fog of war) overlay correctly, so that important areas get maximum artistic quality. | Static maps replace generated maps per zone; dynamic markers overlay correctly on static maps; artistic and generated maps blend in rendering pipeline | F-10.3.14 | R-10.3.14 |
| US-10.3.31 | Designer (P-5) | I want to compose map overlay assets like compass roses, decorative borders, illustrated landmarks, and sea monster illustrations in the visual editor, so that the map has a polished, immersive presentation. | Hand-drawn overlays layer on generated map data; compass roses, borders, and illustrated landmarks supported; overlays composed in visual editor | F-10.3.14 | R-10.3.14 |
| US-10.3.32 | Player (P-23) | I want quest markers to appear simultaneously on the minimap, compass, and world map with consistent icons, so that I always know where my objectives are regardless of which navigation tool I check. | Tracked markers appear on minimap, compass, and world map; consistent icons across all navigation views; untracking a quest hides its markers everywhere | F-10.3.15 | R-10.3.15 |
| US-10.3.33 | Developer (P-15) | I want map markers defined as data assets with icon, color, category, visibility rules, and source type, so that quest objectives, NPCs, resources, party members, enemies, and waypoints are all managed through a unified marker system. | At least 8 marker categories supported; markers defined as data assets with icon, color, category; visibility rules per marker (always, tracked only, zoom range); quest tracker drives marker visibility | F-10.3.15 | R-10.3.15 |
| US-10.3.34 | Player (P-23) | I want overlapping markers to cluster into grouped icons at low zoom levels that expand when I zoom in, so that the map remains readable even in marker-dense areas. | Markers cluster when icons would overlap; clusters expand into individual markers on zoom-in; marker count capped per view for performance | F-10.3.15 | R-10.3.15 |
