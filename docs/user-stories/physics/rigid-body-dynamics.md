1. **US-4.1.1.1** — As a designer (P-5), I want to set physics fixed timestep to 1/60s in the
   project settings, so that physics runs at consistent 60Hz.
   - **Acceptance:** —
2. **US-4.1.1.2** — As an engine tester (P-27), I want to run identical 1000-frame simulations on 3
   platforms and compare state, so that determinism is verified.
   - **Acceptance:** —
3. **US-4.1.1.3** — As a player (P-23), I want physics to behave identically at 30fps and 144fps, so
   that gameplay is fair regardless of hardware.
   - **Acceptance:** —
4. **US-4.1.1.4** — As an engine developer (P-26), I want to implement symplectic Euler integration
   with fixed dt, so that energy conservation is maintained.
   - **Acceptance:** —
5. **US-4.1.1.5** — As an engine developer (P-26), I want to implement Verlet integration as an
   alternative method, so that position-based constraints are more stable.
   - **Acceptance:** —
6. **US-4.1.1.6** — As an engine developer (P-26), I want to disable fast-math compiler
   optimizations on all platforms, so that IEEE 754 compliance is guaranteed.
   - **Acceptance:** —
7. **US-4.1.1.7** — As an engine tester (P-27), I want to record and replay a physics simulation and
   get bit-identical results, so that bugs are reproducible from saved replays.
   - **Acceptance:** —
8. **US-4.1.1.8** — As a player (P-23), I want thrown objects to follow the same arc every time, so
   that I can aim reliably.
