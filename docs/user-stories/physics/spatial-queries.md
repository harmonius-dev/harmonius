1. **US-4.4.1.1** — As a game developer (P-15), I want to cast a ray and receive the hit entity,
   position, normal, and distance, so that line-of-sight and targeting queries work.
   - **Acceptance:** —
2. **US-4.4.1.2** — As an engine developer (P-26), I want to implement RayCast as a system parameter
   that queries the shared spatial index, so that any system can issue ray casts.
   - **Acceptance:** —
3. **US-4.4.1.3** — As a game developer (P-15), I want ray casts to filter by CollisionLayers via
   QueryFilter, so that only relevant entities are hit.
   - **Acceptance:** —
4. **US-4.4.1.4** — As a player (P-23), I want weapons to hit exactly where I aim, so that combat
   targeting feels precise.
   - **Acceptance:** —
5. **US-4.4.1.5** — As an engine tester (P-27), I want to verify ray cast results match expected hit
   position and normal for known geometry, so that accuracy is validated.
   - **Acceptance:** —
6. **US-4.4.1.6** — As a game developer (P-15), I want to use ray casts for line-of-sight between
   entities, so that AI visibility and stealth mechanics work.
   - **Acceptance:** —
7. **US-4.4.1.7** — As a designer (P-5), I want to set maximum ray cast distance per query, so that
   queries do not waste time on distant geometry.
   - **Acceptance:** —
8. **US-4.4.1.8** — As an engine developer (P-26), I want ray casts to traverse the shared BVH
   spatial index, so that physics, rendering, and AI share the same acceleration structure.
   - **Acceptance:** —
9. **US-4.4.1.9** — As an engine tester (P-27), I want to test ray casts against box, sphere,
   capsule, convex, trimesh, and heightfield shapes, so that all shape intersections work.
   - **Acceptance:** —
10. **US-4.4.1.10** — As a game developer (P-15), I want ray cast results to include the surface
    normal at the hit point, so that I can orient decals and effects.
    - **Acceptance:** —
11. **US-4.4.1.11** — As a game developer (P-15), I want ray cast results to include the hit
    entity's CollisionLayers, so that I know what type of surface was hit.
    - **Acceptance:** —
12. **US-4.4.1.12** — As a player (P-23), I want the targeting reticle to align with the actual hit
    point, so that aiming feels responsive and trustworthy.
    - **Acceptance:** —
