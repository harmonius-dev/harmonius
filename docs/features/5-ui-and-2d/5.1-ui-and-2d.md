# 5.1 UI and 2D

## F-5.1.1 Vector-Based Retained UI

GPU-accelerated vector path rendering (fill and stroke) using a compute-driven pipeline. Supports Bezier curves, gradients, rounded rectangles, and resolution-independent shapes. Uses a retained scene tree with dirty flags for incremental updates.

- **Requirements:** [R-2.2.7](../../requirements/2-nonfunctional/2.2-hardware.md) 64-bit atomics (capability-gated)

## F-5.1.2 Bitmap-Based Retained UI

Atlas-batched quad rendering for bitmap UI elements (icons, panels, HUD). Quads are packed per atlas page and drawn via the mesh shader pipeline with alpha blending. A layout engine computes Flexbox-like positioning.

- **Requirements:** [R-2.2.1](../../requirements/2-nonfunctional/2.2-hardware.md) bindless

## F-5.1.3 2D Game Support (Sprites, Tilemaps)

Sprite rendering with per-instance position, rotation, scale, UV rect, atlas index, tint, and z-order. Tilemap rendering with chunked tile grids mapped to atlas UVs via a compute step. Supports multiple layers for parallax.

## F-5.1.4 2.5D Game Support (Isometric, Parallax)

Isometric tile rendering with CPU-side depth sorting (row + col + height key). Parallax layer compositing with depth-based UV scrolling for side-scrolling and isometric games.
