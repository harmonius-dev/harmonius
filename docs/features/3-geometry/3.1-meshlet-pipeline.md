# 3.1 Meshlet Pipeline

## F-3.1.1 Meshlet-Based Virtualized Geometry

All geometry is decomposed into meshlets (small clusters of ~128 vertices and ~128 triangles) stored in a hierarchical cluster structure. Meshlets are the atomic unit of GPU culling, LOD selection, and streaming. This enables fine-grained visibility determination and bandwidth-efficient rendering.

- **Requirements:** [1.1.3](../../requirements/1-architecture/1.1-core-constraints.md) mesh shaders, [2.2.2](../../requirements/2-nonfunctional/2.2-hardware.md) mesh shaders required, [2.2.1](../../requirements/2-nonfunctional/2.2-hardware.md) bindless

## F-3.1.2 Mesh Shader Pipeline (Task/Mesh)

Task (object) shaders perform per-meshlet culling and LOD selection. Surviving meshlets are dispatched to mesh shaders that output triangles directly to the rasterizer. Fragment shaders produce final pixel output. No traditional vertex/index pipeline is used.

- **Requirements:** [1.1.3](../../requirements/1-architecture/1.1-core-constraints.md) mesh shaders, [1.1.7](../../requirements/1-architecture/1.1-core-constraints.md) no legacy pipelines

## F-3.1.3 GPU-Evaluated Spline Paths

Catmull-Rom / cubic Bezier splines evaluated in the task shader. The spline can be extruded into tube geometry (roads, rails, pipes) rendered via the mesh shader with proper UV mapping and LOD.

- **Requirements:** [1.1.3](../../requirements/1-architecture/1.1-core-constraints.md) mesh shaders
