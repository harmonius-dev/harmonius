# 3.1 Meshlet Pipeline

## F-3.1.1 Meshlet-Based Virtualized Geometry

All geometry is decomposed into meshlets (small clusters of ~128 vertices and ~128 triangles) stored in a hierarchical cluster structure. Meshlets are the atomic unit of GPU culling, LOD selection, and streaming. This enables fine-grained visibility determination and bandwidth-efficient rendering.

- **Requirements:** [R-1.1.3](../../requirements/1-architecture/1.1-core-constraints.md) mesh shaders, [R-3.2.2](../../requirements/3-nonfunctional/3.2-hardware.md) mesh shaders required, [R-3.2.1](../../requirements/3-nonfunctional/3.2-hardware.md) bindless

## F-3.1.2 Mesh Shader Pipeline (Task/Mesh)

Task (object) shaders perform per-meshlet culling and LOD selection. Surviving meshlets are dispatched to mesh shaders that output triangles directly to the rasterizer. Fragment shaders produce final pixel output. No traditional vertex/index pipeline is used.

- **Requirements:** [R-1.1.3](../../requirements/1-architecture/1.1-core-constraints.md) mesh shaders, [R-1.1.7](../../requirements/1-architecture/1.1-core-constraints.md) no legacy pipelines

## F-3.1.3 GPU-Evaluated Spline Paths

Catmull-Rom / cubic Bezier splines evaluated in the task shader. The spline can be extruded into tube geometry (roads, rails, pipes) rendered via the mesh shader with proper UV mapping and LOD.

- **Requirements:** [R-1.1.3](../../requirements/1-architecture/1.1-core-constraints.md) mesh shaders

## F-3.1.4 Dynamic Tessellation and Displacement

Runtime adaptive tessellation generating triangles to match displacement detail at the current pixel density. Displacement maps or procedural materials drive vertex offsets on dynamically generated sub-triangles, adding geometric detail beyond the base mesh without offline baking.

- **Requirements:** [R-1.1.3](../../requirements/1-architecture/1.1-core-constraints.md) mesh shaders

## F-3.1.5 Visibility Buffer Rendering

64-bit visibility buffer rasterization storing triangle and instance IDs per pixel via atomic operations. Material evaluation is deferred to a fullscreen pass that reads only the visible pixels, eliminating redundant shading and reducing bandwidth versus a traditional G-buffer for high-polygon scenes.

- **Requirements:** [R-1.1.3](../../requirements/1-architecture/1.1-core-constraints.md) mesh shaders, [R-3.2.1](../../requirements/3-nonfunctional/3.2-hardware.md) bindless

## F-3.1.6 Variable Rate Shading (VRS)

Compute-shader-driven variable rate shading that reduces fragment shading rate in perceptually less important screen regions. A per-tile shading rate image is generated based on luminance variance, motion, and edge detection, then applied during rasterization to trade shading quality for performance.
