# 3.2 Worlds and Terrain

## F-3.2.1 Infinite Streamable Universes

Chunk-based world streaming with configurable chunk size, LOD levels, and resident pool capacity. Chunks stream in/out based on camera distance. The residency system tracks which chunks are loaded, loading, or evicted.

- **Requirements:** [R-3.1.6](../../requirements/3-nonfunctional/3.1-performance.md) priority streaming, [R-3.2.4](../../requirements/3-nonfunctional/3.2-hardware.md) transfer queues, [R-3.2.6](../../requirements/3-nonfunctional/3.2-hardware.md) sparse textures

## F-3.2.2 Planetary-Scale Voxel Worlds

Sparse voxel octree traversal on the GPU with isosurface extraction (dual contouring) in the task/mesh shader. Voxel pages are streamed on demand using sparse 3D textures.

- **Requirements:** [R-3.2.6](../../requirements/3-nonfunctional/3.2-hardware.md) sparse textures, [R-1.1.3](../../requirements/1-architecture/1.1-core-constraints.md) mesh shaders

## F-3.2.3 Detailed Open-World Terrains

Heightmap terrain with tile-based streaming, virtual texturing for surface materials, and LOD via the meshlet pipeline. Height tiles are streamed from disk; a page table tracks virtual texture residency.

- **Requirements:** [R-3.1.6](../../requirements/3-nonfunctional/3.1-performance.md) priority streaming, [R-3.2.6](../../requirements/3-nonfunctional/3.2-hardware.md) sparse textures

## F-3.2.4 Procedural Asset Generation

GPU-side procedural geometry generation via async compute. Jobs produce meshlets written to a ring buffer output slot, fenced to the transfer timeline. Supports noise-based heightmaps, SDF extraction, and parametric shapes.

- **Requirements:** [R-3.2.4](../../requirements/3-nonfunctional/3.2-hardware.md) async compute

## F-3.2.5 Hierarchical Level of Detail (HLOD)

Automatic generation of proxy meshes that combine multiple static mesh instances into simplified cluster representations. At distance, the cluster proxy replaces all constituent objects, reducing draw calls and memory. HLOD generation is grid-cell-aligned for streaming worlds.

## F-3.2.6 Landscape Material Layers

Multi-layer terrain material blending with per-layer weight maps painted or procedurally generated. Each layer is a full material (albedo, normal, roughness) composited at the pixel level using height-weighted blending for natural transitions.

## F-3.2.7 Virtual Heightfield Mesh

Runtime tessellation of heightfield surfaces driven by runtime virtual textures. An endlessly subdividing mesh reads height data from a virtual texture, providing displacement detail without traditional tessellation hardware, suitable for landscape overlays and procedural terrain features.
