# 3.5 — Sky & Atmosphere

## Procedural Sky

| ID | Feature | Description | Requirements | Dependencies | Platform Notes |
|----|---------|-------------|--------------|--------------|----------------|
| F-3.5.1 | Procedural Sky Model | A physically-motivated analytical sky model (Preetham/Hosek-Wilkie) provides fast sky color evaluation for any sun position. The model computes sky luminance and chromaticity as a function of view angle, sun zenith, and atmospheric turbidity. It serves as the fallback sky representation when volumetric atmosphere scattering is budget-limited, and as the reference for baking sky lookup tables. | R-3.5.1 | None | Analytical sky model is lightweight and runs at full quality on all platforms. Used as the default sky on mobile when volumetric atmosphere is disabled. |

## Physically-Based Atmosphere Scattering

| ID | Feature | Description | Requirements | Dependencies | Platform Notes |
|----|---------|-------------|--------------|--------------|----------------|
| F-3.5.2 | Multi-Scattering Atmosphere with Aerial Perspective | A precomputed atmosphere scattering model based on Bruneton's improved method. Rayleigh scattering, Mie scattering, and ozone absorption are integrated along view rays using precomputed transmittance, single-scattering, and multi-scattering lookup tables. The LUTs are recomputed when atmosphere parameters change. Aerial perspective is applied to all scene geometry via a froxel-based volume, fading distant terrain and structures into atmospheric haze naturally. This is critical for MMO draw distances spanning tens of kilometers. | R-3.5.2 | None | LUT resolution scales per tier: mobile uses lower-res LUTs with fewer scattering samples. Froxel volume depth slices reduced on mobile (16 vs 64). |

## Volumetric Clouds

| ID | Feature | Description | Requirements | Dependencies | Platform Notes |
|----|---------|-------------|--------------|--------------|----------------|
| F-3.5.3 | Ray-Marched Volumetric Clouds | Clouds are rendered by ray marching through a cloud volume defined by layered noise fields (Perlin-Worley for shape, Worley for detail erosion). Cloud density, coverage, and type are controlled by a weather map texture that can be animated or driven by a weather simulation. Lighting uses a cone-sampled in-scattering approach with multi-octave extinction and ambient sky contribution. Temporal reprojection across multiple frames amortizes the per-pixel ray-march cost. | R-3.5.3 | F-3.5.2 | Ray-march step count scales per tier: mobile 32-48 steps, Switch 64, desktop 96-128. Temporal reprojection essential on all platforms; mobile reprojects over more frames (4-6 vs 2 on desktop). |
| F-3.5.4 | Cloud Shadow Map | Cloud coverage is projected onto the terrain and scene as a shadow map generated by rendering the cloud density field from the sun's perspective. This map modulates direct lighting on terrain, foliage, and water, producing large-scale moving shadow patterns across the open world that reinforce the sense of dynamic weather and scale. | R-3.5.4 | F-3.5.3 | Cloud shadow map resolution scales per tier: mobile 512x512, desktop 2048x2048. Update rate reduced on mobile (every 2-4 frames). |

## Dynamic Time of Day

| ID | Feature | Description | Requirements | Dependencies | Platform Notes |
|----|---------|-------------|--------------|--------------|----------------|
| F-3.5.5 | Dynamic Time-of-Day System | A time-of-day controller drives sun and moon positions along astronomically-derived arcs, continuously updating sky colors, atmosphere LUTs, ambient light intensity, shadow direction, and fog parameters. Transitions through dawn, day, dusk, and night are smooth and physically motivated. The system exposes a time scale parameter for gameplay control, supporting accelerated day/night cycles common in MMO worlds. | R-3.5.5 | F-3.5.2, F-3.5.3 | Time-of-day parameter updates are lightweight on all platforms. LUT rebuild frequency reduced on mobile (triggered only on significant sun angle change). |

## Celestial Bodies

| ID | Feature | Description | Requirements | Dependencies | Platform Notes |
|----|---------|-------------|--------------|--------------|----------------|
| F-3.5.6 | Celestial Body Rendering | Sun, moon, stars, and planetary bodies are rendered as part of the sky dome. The sun is a bright disc with configurable limb darkening. The moon uses a textured disc with phase-accurate illumination. Stars are rendered from a catalog with magnitude-based brightness, twinkling, and atmospheric extinction near the horizon. All celestial positions are astronomically computed or artist-overridden. | R-3.5.6 | F-3.5.5 | Star count reduced on mobile (fewer catalog entries rendered). Twinkling animation disabled on mobile. Moon uses lower-res texture on mobile. |

## Sky Capture

| ID | Feature | Description | Requirements | Dependencies | Platform Notes |
|----|---------|-------------|--------------|--------------|----------------|
| F-3.5.7 | Environment Cubemap Capture | The sky, atmosphere, and cloud systems are captured into a low-resolution environment cubemap each frame or at reduced update rates. This cubemap provides ambient diffuse and specular lighting for the entire scene via image-based lighting (IBL). Capture is performed by rendering the sky shader into cubemap faces on a round-robin schedule to amortize cost, ensuring reflections and ambient lighting always match the current sky state. | R-3.5.7 | F-3.5.2, F-3.5.3, F-3.5.6 | Cubemap capture resolution scales per tier: mobile 64x64, Switch 128x128, desktop 256x256. Update rate: mobile every 4-8 frames, desktop every 1-2. |
