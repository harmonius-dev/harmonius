# 4.1 Animation

## F-4.1.1 Weighted Skeletal Animation (GPU Skinning)

GPU compute skinning using bone palettes of world-space matrices. Supports dual-quaternion blending to reduce candy-wrapping artifacts at joints.

- **Requirements:** [R-2.3.2](../../requirements/2-nonfunctional/2.3-data-constraints.md) 256 bone limit, [R-2.3.12](../../requirements/2-nonfunctional/2.3-data-constraints.md) pipeline ordering

## F-4.1.2 Instanced Skeletal Animation

Batch evaluation of animation for thousands of instances in a single compute dispatch using an arena buffer. Enables large crowds and armies without per-instance CPU overhead.

- **Requirements:** [R-2.2.4](../../requirements/2-nonfunctional/2.2-hardware.md) async compute

## F-4.1.3 Morph Shapes (Blend Shapes)

GPU compute accumulation of weighted morph target deltas (position and normal offsets) applied before skinning. Supports facial animation, damage deformation, and corrective shapes.

- **Requirements:** [R-2.3.5](../../requirements/2-nonfunctional/2.3-data-constraints.md) f16 precision, [R-2.3.12](../../requirements/2-nonfunctional/2.3-data-constraints.md) morph before skinning

## F-4.1.4 Animation Clips (GPU Keyframe Evaluation)

Hermite-interpolated animation curve evaluation on the GPU. Clips define per-joint translation, rotation, and scale tracks sampled at arbitrary time offsets. Supports looping and clamped playback.

- **Requirements:** [R-2.2.4](../../requirements/2-nonfunctional/2.2-hardware.md) async compute

## F-4.1.5 Animation State Machines

CPU-side declarative state machines that evaluate transitions based on named boolean, float, and trigger parameters. State machines output blend descriptors (clip weights and times) uploaded to the GPU each frame.

- **Requirements:** [R-1.1.1](../../requirements/1-architecture/1.1-core-constraints.md) safe user API, [R-2.3.3](../../requirements/2-nonfunctional/2.3-data-constraints.md) 4 blend clips

## F-4.1.6 Procedural Animation (IK, Ragdoll)

GPU compute inverse kinematics (FABRIK solver) for end-effector goals and pole vectors. Position-based dynamics ragdoll simulation on async compute, with results patching the relevant bones in the skeleton. Ragdoll readback to CPU is opt-in.

- **Requirements:** [R-2.3.4](../../requirements/2-nonfunctional/2.3-data-constraints.md) 8 bone IK limit

## F-4.1.7 Custom Animation Curves

User-defined Hermite-interpolated curves with arbitrary output semantics (float values controlling material properties, camera FOV, etc.). Evaluated CPU-side within the state machine.

## F-4.1.8 Cloth Simulation

GPU position-based dynamics cloth with distance, bending, and self-collision constraints. Cloth meshes are driven by wind forces, character animation, and collision capsules. A panel-based authoring model allows non-destructive constraint editing.

- **Requirements:** [R-2.2.4](../../requirements/2-nonfunctional/2.2-hardware.md) async compute

## F-4.1.9 Destruction and Fracture

Real-time geometry fracturing using pre-fractured geometry collections with hierarchical cluster groupings. Runtime forces break cluster connections, spawning independent rigid body fragments. Integrates with the particle system for debris and dust emission.

## F-4.1.10 Hair and Fur Simulation

Physics-based strand simulation using guide curves that drive interpolated strands via skinning. Supports wind, gravity, and collision with skeletal mesh capsules. LOD transitions between full strand simulation at close range and simplified card-based representation at distance.
