# 6.1 Shader and Asset Pipeline

## F-6.1.1 Data-Driven Shader Graph Authoring

Visual shader graphs defined as DAGs of typed nodes connected by typed edges. Graphs are serialized to a compact binary format. Each graph can declare permutation axes for compile-time branching (e.g., alpha mode, quality tier).

- **Requirements:** [R-2.3.8](../../requirements/2-nonfunctional/2.3-data-constraints.md) single shader IR, [R-2.3.9](../../requirements/2-nonfunctional/2.3-data-constraints.md) binary format

## F-6.1.2 Shader Graph Compilation

A standalone compiler transforms shader graphs into platform-native shader code via a platform-neutral intermediate representation. Supports validation (type checking, cycle detection), permutation enumeration, reflection data extraction, and cross-compilation to all target backends.

- **Requirements:** [R-2.3.8](../../requirements/2-nonfunctional/2.3-data-constraints.md) single shader IR

## F-6.1.3 Custom Shader Nodes

Third-party shader node types can be registered via a node registry, providing a slot signature and IR lowering function. This enables engine-specific or game-specific shader operations without modifying the compiler.

## F-6.1.4 Resource CPU Readback

Typed asynchronous readback of GPU buffer data to CPU memory via staging buffers and fence synchronization. Used for debugging, screenshots, procedural data persistence, and physics feedback.

- **Requirements:** [R-2.2.4](../../requirements/2-nonfunctional/2.2-hardware.md) transfer queues

## F-6.1.5 3D Asset Import (glTF / Blender)

Import pipeline for glTF 2.0 assets: mesh extraction, meshletization, texture loading into the bindless heap, and automatic PBR material-to-shader-graph conversion.

- **Requirements:** [R-2.3.10](../../requirements/2-nonfunctional/2.3-data-constraints.md) glTF import format
