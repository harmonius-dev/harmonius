# 6.2 IO and Streaming

## F-6.2.1 Transfer Queue Pool

A pool of dedicated transfer queue slots for background GPU uploads. Each slot manages staging memory, command recording, and in-flight upload tracking independently.

- **Requirements:** [R-3.2.4](../../requirements/3-nonfunctional/3.2-hardware.md) transfer queues, [R-3.1.5](../../requirements/3-nonfunctional/3.1-performance.md) fixed-capacity pools

## F-6.2.2 Terrain and Texture Tile Streaming

Priority-based streaming of 2D tiles (terrain heightmaps, virtual texture pages) from disk to GPU. A page table tracks residency. Page faults trigger streaming requests; resolved pages appear in the next available frame.

- **Requirements:** [R-3.1.6](../../requirements/3-nonfunctional/3.1-performance.md) priority streaming, [R-3.2.6](../../requirements/3-nonfunctional/3.2-hardware.md) sparse textures

## F-6.2.3 3D Texture Streaming (Voxel)

Streaming of 3D texture slices for voxel world data using sparse 3D textures and a residency map.

- **Requirements:** [R-3.2.6](../../requirements/3-nonfunctional/3.2-hardware.md) sparse textures

## F-6.2.4 Platform-Native Direct IO

File-to-GPU DMA using each platform's native high-performance IO path, bypassing the CPU for bulk asset transfers. Completion is signaled via GPU fences.

- **Requirements:** [R-1.2.4](../../requirements/1-architecture/1.2-platform-support.md) platform-native IO

## F-6.2.5 Async Compute Integration

Multi-frame GPU compute jobs (terrain generation, voxel meshing, GI probe updates, acceleration structure builds) dispatched to a dedicated compute queue, overlapping with graphics work.

- **Requirements:** [R-3.2.4](../../requirements/3-nonfunctional/3.2-hardware.md) async compute, [R-3.1.7](../../requirements/3-nonfunctional/3.1-performance.md) triple-buffered pipeline

## F-6.2.6 Resource Synchronization

Explicit fence-based synchronization across graphics, compute, and transfer queues. All resource transitions are driven by the render graph compiler's barrier insertion pass.

- **Requirements:** [R-3.1.2](../../requirements/3-nonfunctional/3.1-performance.md) minimal barriers, [R-3.2.5](../../requirements/3-nonfunctional/3.2-hardware.md) timeline fences

## F-6.2.7 Split-Screen and Multi-Window

Multi-camera rendering with shared resources (shadow maps, GI probes, meshlet buffers) and per-camera exclusive resources (render targets, per-camera uniforms). Each camera can be assigned to a separate render thread.

- **Requirements:** [R-3.1.1](../../requirements/3-nonfunctional/3.1-performance.md) parallel encoding, [R-3.3.7](../../requirements/3-nonfunctional/3.3-data-constraints.md) single-writer invariant

## F-6.2.8 GPU Decompression (DirectStorage)

GPU-side decompression of compressed asset data streamed from SSD, bypassing CPU decompression entirely. Compressed textures and geometry are transferred directly from storage to GPU memory where a compute shader decompresses them in-place, maximizing IO throughput and minimizing CPU overhead.

## F-6.2.9 Bindless Resource Management

Unified bindless descriptor heap managing all textures, buffers, and samplers via integer indices rather than per-draw-call descriptor set bindings. Materials reference resources by index into the global heap, eliminating per-shader texture binding limits and enabling arbitrary material combinations in a single draw call.
