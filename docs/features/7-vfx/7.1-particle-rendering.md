# 7.1 Particle and VFX Rendering

## F-7.1.1 GPU Particle Simulation

Compute-shader particle simulation supporting millions of particles. Each emitter defines spawn rate, lifetime, and a chain of simulation modules (forces, noise, collision, drag) evaluated per-particle per-frame on the GPU. Particles are stored in a persistent GPU buffer with free-list allocation.

- **Requirements:** [R-3.2.4](../../requirements/3-nonfunctional/3.2-hardware.md) async compute

## F-7.1.2 Sprite Particle Rendering

Camera-facing billboard particle rendering with atlas-based flipbook animation. Each particle samples a sprite sheet sub-region with configurable animation rate, blend modes (additive, alpha, premultiplied), and per-particle color/size/rotation curves.

## F-7.1.3 Mesh Particle Rendering

Instanced 3D mesh rendering driven by particle state. Each particle drives a mesh instance with full material support, per-particle transform, scale, and material parameter overrides. Supports LOD selection based on particle distance.

## F-7.1.4 Ribbon and Trail Rendering

Spline-based ribbon geometry connecting sequential particles sharing a trail ID. Ribbon meshes are generated per-frame from particle positions with configurable width, twist, UV tiling, and tessellation. Used for sword trails, magic effects, and smoke trails.

## F-7.1.5 Particle Light Emission

Dynamic point light sources spawned from particle positions. Each emitting particle contributes a light with configurable color, intensity, and attenuation radius to the scene's light buffer. Compatible with stochastic many-light sampling for large particle counts.

## F-7.1.6 GPU Fluid Simulation

Grid-based Eulerian fluid simulation on the GPU for fire, smoke, and liquid surface effects. A 3D velocity/density/temperature grid is advected via compute shaders. The resulting density field is ray-marched for volumetric rendering with emission, absorption, and scattering.

- **Requirements:** [R-3.2.4](../../requirements/3-nonfunctional/3.2-hardware.md) async compute

## F-7.1.7 Particle Collision

GPU-driven particle collision against the scene depth buffer (screen-space) and signed distance fields (world-space). Particles bounce, stick, or die on contact based on configurable restitution and friction parameters. SDF collision enables off-screen and volumetric obstacle interaction.

## F-7.1.8 Particle Data Interfaces

Typed data channels allowing particles to read from and write to external data sources: skeletal mesh surfaces (spawn on mesh), textures (sample color/velocity fields), curves (parameter animation), and other emitters (particle-to-particle interaction).

## F-7.1.9 Destruction System

Real-time destruction using volume distance fields for destruction masking and fracture pattern generation. Meshes are pre-fractured into chunks with configurable granularity; runtime damage events activate chunk separation with physics simulation. Destruction dynamically modifies terrain geometry, texturing, and environmental rendering. Supports cascading structural failure and debris particle spawning.
