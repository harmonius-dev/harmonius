# Test Case Coverage Audit

Tracks gaps where R-X.Y.Z or US-X.Y.Z IDs referenced in a design document are not covered by any TC
entry in the companion test-cases file.

Coverage rule: every R-ID and US-ID referenced in a design must appear in at least one TC
requirement-row in its companion file. Performance requirements (R-X.Y.Z with a/b/c suffix) must
additionally have a Benchmark TC (TC-X.Y.Z.B*).

Generated by an inline bash audit script (see plan velvet-chasing-canyon.md).
