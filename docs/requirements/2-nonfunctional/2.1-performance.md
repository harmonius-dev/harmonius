# 2.1 Performance

## 2.1.1 Parallel Command Encoding

Command encoding SHALL be parallelized across multiple threads to saturate the GPU command processor. Each encoding thread SHALL operate on an independent command buffer with no shared mutable state.

## 2.1.2 Minimal Barriers

The render graph compiler SHALL insert the minimum set of memory and execution barriers necessary for correctness. Redundant barriers SHALL NOT be inserted. This minimizes GPU pipeline stalls.

## 2.1.3 Resource Lifetime Aliasing

The render graph compiler SHALL compute resource lifetimes and allow resources with non-overlapping lifetimes to share the same GPU memory. This reduces peak VRAM usage for transient render targets.

## 2.1.4 Feature and Budget Gates

Render passes SHALL support both static capability gates (compile-time, based on GPU features) and dynamic budget gates (runtime, based on frame time). Passes that fail their gate SHALL be culled from the execution plan, along with any exclusively-dependent passes.

## 2.1.5 Fixed-Capacity Resource Pools

GPU resource pools SHALL have fixed capacity to prevent runtime fragmentation. Pool sizes SHALL be determined at initialization.

## 2.1.6 Streaming with Priority Scheduling

Asset streaming SHALL use priority-based scheduling (critical, high, normal, low) with camera-based predictive prefetch. Eviction SHALL use LRU with hysteresis to prevent thrashing.

## 2.1.7 Triple-Buffered Frame Pipeline

The frame pipeline SHALL support triple-buffering to overlap CPU graph building, GPU command encoding, and GPU execution. Frame synchronization SHALL use timeline fences.

## 2.1.8 Zero-Allocation Execution Path

The per-frame execution hot path SHALL not perform heap allocations. All per-frame data SHALL flow through pre-allocated ring buffers and staging memory.

## 2.1.9 Quality Tier Fallback Chains

Rendering features SHALL support tiered quality fallbacks (e.g., RT soft shadows → PCSS → PCF) controlled by capability flags and budget gates. This enables graceful degradation across hardware tiers.
