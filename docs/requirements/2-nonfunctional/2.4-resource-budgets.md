# 2.4 Resource Budgets

## R-2.4.1 Meshlet Geometry Limits

Each meshlet SHALL contain at most 128 vertices and 128 triangles. Meshlets exceeding this limit SHALL be split during asset import. This ensures uniform workgroup occupancy in mesh shaders and predictable GPU scheduling.

## R-2.4.2 Active Light Capacity

The forward+ path SHALL support at least 1,024 active point, spot, and directional lights per frame. The clustered/tiled light assignment grid SHALL have a configurable tile size (default: 16x16 pixels). The deferred path SHALL support at least the same light count via fullscreen compute accumulation.

## R-2.4.3 Shadow Atlas Budget

Shadow maps SHALL be rendered into a shared atlas with configurable total resolution (default: 8192x8192). Individual shadow map tiles SHALL use power-of-two resolutions from 256 to 2048 pixels, assigned by light priority and screen-space contribution.

## R-2.4.4 Streaming Pool Capacity

The asset streaming system SHALL maintain a configurable GPU memory pool per resource type (terrain tiles, virtual texture pages, voxel pages, meshlet clusters). Combined default capacity SHALL be at least 2 GB. Pool exhaustion SHALL trigger LRU eviction per R-2.1.6.

## R-2.4.5 DDGI Probe Grid

The DDGI probe grid SHALL support configurable probe spacing and grid dimensions up to 64x32x64 probes. Irradiance texels per probe SHALL default to 8x8 and visibility texels SHALL default to 16x16, stored in octahedral atlas mapping.

## R-2.4.6 Froxel Grid Dimensions

The volumetric fog froxel grid SHALL have a configurable XY tile size (default: 8x8 pixels) and depth slice count (default: 128) with exponential depth distribution for near-field density.

## R-2.4.7 Sprite and Tile Batch Capacity

The 2D renderer SHALL support at least 100,000 sprite instances per draw call and at least 65,536 visible tile cells per tilemap layer, both via GPU instancing.

## R-2.4.8 UI Atlas Configuration

The bitmap UI system SHALL use atlas pages of configurable size (default: 4096x4096 RGBA8). Atlas growth SHALL be bounded by a platform-defined maximum page count.

## R-2.4.9 Virtual Texture Page Size

Virtual texture pages SHALL be 128x128 or 256x256 texels (configurable per texture). The page table indirection texture SHALL support at least 16K x 16K logical texture resolution.
