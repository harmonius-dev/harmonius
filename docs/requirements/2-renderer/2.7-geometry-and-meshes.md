# 2.7 Geometry and Meshes

## R-2.7.1 Add Mesh Entities

The system SHALL accept mesh resource handles from the asset import pipeline and automatically decompose them into meshlets for GPU-driven rendering. Mesh entities SHALL be added to the scene via the declarative API.

- **Features:** [F-3.1.1](../../features/3-geometry/3.1-meshlet-pipeline.md) meshlet-based virtualized geometry

## R-2.7.2 Configure Meshlet LOD

The system SHALL support per-asset LOD bias and transition distance configuration. The meshlet hierarchy SHALL select the appropriate LOD level based on screen-space size.

- **Features:** [F-3.1.1](../../features/3-geometry/3.1-meshlet-pipeline.md) meshlet-based virtualized geometry

## R-2.7.3 Instance Mesh Entities

The system SHALL support instancing mesh entities that share geometry but have independent transforms and material overrides. Instances SHALL be compacted and dispatched via indirect draw.

- **Features:** [F-1.1.7](../../features/1-rendering/1.1-core-rendering.md) GPU-driven instancing

## R-2.7.4 Create GPU Splines

The system SHALL support Catmull-Rom and cubic Bezier spline paths evaluated in the task shader. Splines SHALL be extrudable into tube geometry with proper UV mapping and LOD.

- **Features:** [F-3.1.3](../../features/3-geometry/3.1-meshlet-pipeline.md) GPU-evaluated spline paths

## R-2.7.5 Submit Procedural Geometry Jobs

The system SHALL accept GPU compute jobs that produce meshlet geometry written to a ring buffer output slot. Jobs SHALL be fenced to the transfer timeline for safe consumption.

- **Features:** [F-3.2.4](../../features/3-geometry/3.2-worlds-and-terrain.md) procedural asset generation

## R-2.7.6 Configure Mesh Shader Debug Viz

The system SHALL provide debug visualization overlays for meshlet boundaries, LOD levels, and culling results. Debug modes SHALL be toggleable at runtime.

- **Features:** [F-3.1.2](../../features/3-geometry/3.1-meshlet-pipeline.md) mesh shader pipeline

## R-2.7.7 Query Meshlet Statistics

The system SHALL expose per-frame statistics for submitted meshlets, survived meshlets after culling, and dispatch counts. Statistics SHALL be queryable via the diagnostic API.

## R-2.7.8 Configure Instanced Draw Batching

The system SHALL batch opaque meshlet instances by shared material and material instance into indirect draw calls.
The maximum instances per indirect draw call SHALL be configurable. Material-based batching statistics SHALL be
observable for performance tuning. Opaque meshlets SHALL NOT be depth-sorted. Transparent objects SHALL NOT be
batched — they SHALL be drawn individually in sorted order (see [R-2.4.6](2.4-materials-and-shading.md)).

## R-2.7.9 Configure Dynamic Tessellation

The system SHALL support runtime adaptive tessellation with displacement maps, generating sub-triangles to match pixel-density-driven subdivision levels. Tessellation SHALL operate within the mesh shader pipeline.

- **Features:** [F-3.1.4](../../features/3-geometry/3.1-meshlet-pipeline.md) dynamic tessellation and displacement

## R-2.7.10 Enable Visibility Buffer

The system SHALL support 64-bit visibility buffer rasterization storing triangle and instance IDs per pixel. Material evaluation SHALL be deferred to a fullscreen pass reading only visible pixels.

- **Features:** [F-3.1.5](../../features/3-geometry/3.1-meshlet-pipeline.md) visibility buffer rendering

## R-2.7.11 Configure Variable Rate Shading

The system SHALL support compute-driven per-tile shading rate images based on luminance variance, motion, and edge detection. Shading rate SHALL be applied during rasterization to reduce fragment shading in perceptually less important regions.

- **Features:** [F-3.1.6](../../features/3-geometry/3.1-meshlet-pipeline.md) variable rate shading

## R-2.7.12 Enable Meshlet Compression

The system SHALL support lossy compression of meshlet geometry data with configurable per-meshlet quantization of vertex positions, normals, and UVs. Compressed meshlets SHALL be decoded in the mesh shader at runtime.

- **Features:** [F-3.1.7](../../features/3-geometry/3.1-meshlet-pipeline.md) meshlet compression

## R-2.7.13 Enable GPU Work Graphs

The system SHALL support GPU-self-scheduled work dispatch where producer passes emit variable-count work items consumed by downstream passes without host CPU involvement. Work graphs SHALL support recursive expansion/compaction patterns and GPU-driven rendering dispatch chains.

- **Features:** [F-3.1.8](../../features/3-geometry/3.1-meshlet-pipeline.md) GPU work graphs

## R-2.7.14 Enable Concurrent Binary Tree Tessellation

The system SHALL support binary tree-based adaptive tessellation for arbitrary polygon meshes with parallel GPU subdivision and simplification. The system SHALL achieve planetary-scale geometry rendering from coarse input meshes.

- **Features:** [F-3.1.9](../../features/3-geometry/3.1-meshlet-pipeline.md) concurrent binary trees

## R-2.7.15 Configure LOD Generation

The system SHALL support automatic LOD chain generation from source meshes using edge-collapse simplification with quadric error metrics. LOD generation SHALL preserve silhouette edges, UV seams, and material boundaries. Per-level screen-space error bounds SHALL be configurable.

- **Features:** [F-3.1.10](../../features/3-geometry/3.1-meshlet-pipeline.md) automatic LOD generation

## R-2.7.16 Configure LOD Blending

The system SHALL support configurable LOD transition blending with selectable methods per asset: dithered crossfade, screen-door transparency, or vertex morphing. The transition screen-space range SHALL be configurable per asset. LOD blending SHALL be evaluated in the mesh shader pipeline.

- **Features:** [F-3.1.11](../../features/3-geometry/3.1-meshlet-pipeline.md) LOD blending and transitions

## R-2.7.17 Configure Skinned Mesh LOD

The system SHALL support per-skeletal-mesh LOD chains with configurable bone reduction, simplified geometry, and fewer bone influences per vertex at each LOD level. LOD selection SHALL be screen-size-driven. The system SHALL support fallback to rigid impostor or billboard at extreme distance.

- **Features:** [F-3.1.12](../../features/3-geometry/3.1-meshlet-pipeline.md) skinned mesh LOD

## R-2.7.18 Configure Static Mesh Merging

The system SHALL support merging nearby static mesh instances sharing the same material into combined meshlet groups to reduce per-object overhead. Merge candidates SHALL be selected spatially with constraints on material compatibility and maximum merged vertex count. Merging SHALL be reversible at runtime.

- **Features:** [F-3.1.13](../../features/3-geometry/3.1-meshlet-pipeline.md) static mesh merging
