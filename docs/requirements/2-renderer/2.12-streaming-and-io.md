# 2.12 Streaming and IO

## R-2.12.1 Configure Streaming Priorities

The system SHALL support configuring streaming priority levels (critical, high, normal, low) with camera-based predictive prefetch. Priority SHALL influence load ordering and eviction decisions.

- **Features:** [F-6.2.1](../../features/6-tooling-and-io/6.2-io-and-streaming.md) transfer queue pool, [F-6.2.2](../../features/6-tooling-and-io/6.2-io-and-streaming.md) terrain tile streaming

## R-2.12.2 Configure Streaming Pool Budgets

The system SHALL support configuring GPU memory pool capacity per resource type (terrain tiles, virtual texture pages, voxel pages, meshlet clusters). Pool exhaustion SHALL trigger LRU eviction.

## R-2.12.3 Enable Tile Streaming

The system SHALL support priority-based streaming of 2D tiles (terrain heightmaps, virtual texture pages) from disk to GPU. A page table SHALL track residency with fault-driven loading.

- **Features:** [F-6.2.2](../../features/6-tooling-and-io/6.2-io-and-streaming.md) terrain tile streaming

## R-2.12.4 Enable Voxel Streaming

The system SHALL support streaming of 3D texture slices for voxel world data using sparse 3D textures and a residency map.

- **Features:** [F-6.2.3](../../features/6-tooling-and-io/6.2-io-and-streaming.md) 3D texture streaming

## R-2.12.5 Configure Async Compute Jobs

The system SHALL support dispatching multi-frame GPU compute jobs to a dedicated compute queue, overlapping with graphics work.

- **Features:** [F-6.2.5](../../features/6-tooling-and-io/6.2-io-and-streaming.md) async compute integration

## R-2.12.6 Synchronize Cross-Queue Resources

The system SHALL support explicit fence-based synchronization across graphics, compute, and transfer queues. Resource transitions SHALL be driven by the render graph compiler.

- **Features:** [F-6.2.6](../../features/6-tooling-and-io/6.2-io-and-streaming.md) resource synchronization

## R-2.12.7 Configure Split-Screen Views

The system SHALL support multi-camera rendering with shared resources (shadow maps, GI probes, meshlet buffers) and per-camera exclusive resources (render targets, uniforms).

- **Features:** [F-6.2.7](../../features/6-tooling-and-io/6.2-io-and-streaming.md) split-screen and multi-window

## R-2.12.8 Monitor IO Throughput

The system SHALL expose streaming IO throughput metrics including bytes transferred, queue depth, and latency percentiles for performance monitoring.
