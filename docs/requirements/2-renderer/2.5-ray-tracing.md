# 2.5 Ray Tracing

## R-2.5.1 Query RT Hardware Availability

The system SHALL expose a capability descriptor indicating which ray tracing features are supported by the current hardware. Applications SHALL query this before enabling RT effects.

## R-2.5.2 Enable RT Acceleration Structures

The system SHALL automatically build and compact bottom-level acceleration structures (BLAS) from mesh geometry and refit the top-level acceleration structure (TLAS) each frame for dynamic scenes.

- **Features:** [F-2.1.1](../../features/2-advanced-rendering/2.1-ray-tracing.md) acceleration structures

## R-2.5.3 Configure RT Reflections

The system SHALL support hybrid SSR+RT reflections with a configurable roughness threshold determining when RT rays supplement screen-space ray marching.

- **Features:** [F-2.1.2](../../features/2-advanced-rendering/2.1-ray-tracing.md) RT reflections

## R-2.5.4 Select Reflection Fallback

The system SHALL support selecting a reflection fallback mode: SSR-only, cubemap probes, or disabled when RT hardware is unavailable.

## R-2.5.5 Configure RT Sample Counts

The system SHALL allow configuring rays-per-pixel independently for each RT effect (reflections, shadows, GI, AO).

## R-2.5.6 Configure Denoiser Strength

The system SHALL expose per-effect spatial-temporal denoiser aggressiveness controls. Higher aggressiveness SHALL reduce noise at the cost of temporal lag and detail softening.

## R-2.5.7 Mark Entities for RT Participation

The system SHALL support per-entity flags controlling inclusion or exclusion from the RT acceleration structure.

## R-2.5.8 Enable RT Global Illumination

The system SHALL support DDGI-based real-time global illumination with configurable probe update rate and hysteresis for temporal stability.

- **Features:** [F-2.1.4](../../features/2-advanced-rendering/2.1-ray-tracing.md) DDGI

## R-2.5.9 Enable Path Tracing

The system SHALL support a progressive unbiased path tracer with configurable samples per pixel and maximum bounce count. The path tracer SHALL support all material types, volumetrics, and sky atmosphere for ground-truth reference output.

- **Features:** [F-2.1.5](../../features/2-advanced-rendering/2.1-ray-tracing.md) path tracing

## R-2.5.10 Enable RT Subsurface Transmission

The system SHALL support volumetric scattering through translucent media during shadow ray evaluation. When a shadow ray hits a subsurface material, light transport through the material's thickness SHALL be evaluated using the subsurface profile.

- **Features:** [F-2.1.6](../../features/2-advanced-rendering/2.1-ray-tracing.md) RT subsurface transmission

## R-2.5.11 Enable Surfel-Based GI

The system SHALL support dynamic surfel-based global illumination requiring no precomputation. The system SHALL discretize scene geometry into surface elements from the G-Buffer, cache irradiance in a clipmap probe system, and trace rays from surfels via hardware RT to compute incoming radiance with multi-scale temporal estimation.

- **Features:** [F-2.1.7](../../features/2-advanced-rendering/2.1-ray-tracing.md) surfel-based GI

## R-2.5.12 Enable ReSTIR Sampling

The system SHALL support reservoir-based spatiotemporal importance resampling (ReSTIR) for direct illumination (ReSTIR DI) and global illumination (ReSTIR GI). Reservoirs SHALL be maintained per pixel across frames with configurable spatial and temporal reuse parameters.

- **Features:** [F-2.1.8](../../features/2-advanced-rendering/2.1-ray-tracing.md) ReSTIR sampling framework

## R-2.5.13 Enable Real-Time Production Path Tracing

The system SHALL support tiered real-time path tracing with configurable quality levels from direct-RT-only through multi-bounce full path tracing. After the final bounce, the system SHALL fall back to rasterized GI rather than terminating paths. The system SHALL require neural denoising and temporal upscaling for real-time frame rates.

- **Features:** [F-2.1.9](../../features/2-advanced-rendering/2.1-ray-tracing.md) production path tracing

## R-2.5.14 Enable Opacity Micromaps

The system SHALL support per-micro-triangle opacity classification annotations on acceleration structures. Opacity micromaps SHALL accelerate any-hit shader invocations for alpha-tested geometry by classifying micro-triangles as opaque, transparent, or unknown.

- **Features:** [F-2.1.10](../../features/2-advanced-rendering/2.1-ray-tracing.md) opacity micromaps

## R-2.5.15 Enable Shader Execution Reordering

The system SHALL support hardware-assisted reordering of ray tracing shader invocations to group similar shader executions into coherent wavefronts, reducing divergence after intersection testing.

- **Features:** [F-2.1.11](../../features/2-advanced-rendering/2.1-ray-tracing.md) shader execution reordering

## R-2.5.16 Configure Neural Denoising

The system SHALL support AI-trained neural network denoising for ray-traced and path-traced output. The system SHALL allow selecting between neural denoising and conventional spatiotemporal denoisers, with automatic fallback when neural hardware is unavailable.

- **Features:** [F-2.1.12](../../features/2-advanced-rendering/2.1-ray-tracing.md) neural denoising

## R-2.5.17 Enable RT Lens Flare

The system SHALL support physically accurate ray-traced lens flare simulation that traces rays from bright scene sources through modeled lens elements to produce ghost and halo artifacts.

- **Features:** [F-2.1.13](../../features/2-advanced-rendering/2.1-ray-tracing.md) RT lens flare

## R-2.5.18 Enable Voxel-Based GI

The system SHALL support rasterization-only global illumination via world voxelization storing albedo, normals, and translucency. The voxel cache SHALL support re-lighting for time-of-day changes without re-voxelization, combined with irradiance volumes for indirect lighting.

- **Features:** [F-2.1.14](../../features/2-advanced-rendering/2.1-ray-tracing.md) voxel-based GI

## R-2.5.19 Enable Neural Radiance Cache

The system SHALL support a runtime-trained neural network for estimating indirect radiance, enabling early termination of path-traced paths. The network SHALL be continuously trained from a subset of full-length paths while serving cached radiance predictions for terminated paths.

- **Features:** [F-2.1.15](../../features/2-advanced-rendering/2.1-ray-tracing.md) neural radiance cache

## R-2.5.20 Enable Stochastic SSR

The system SHALL support Monte Carlo importance-sampled screen-space reflections with spatiotemporal filtering. Rays SHALL be importance-sampled from the microfacet BRDF to reproduce specular elongation and spatially-varying roughness.

- **Features:** [F-2.1.16](../../features/2-advanced-rendering/2.1-ray-tracing.md) stochastic SSR
