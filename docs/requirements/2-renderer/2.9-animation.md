# 2.9 Animation

## R-2.9.1 Attach Skeleton to Mesh

The system SHALL support attaching skeletons with up to 256 bones to mesh entities. Skinning SHALL support both linear blend and dual-quaternion blending modes.

- **Features:** [F-4.1.1](../../features/4-animation/4.1-animation.md) weighted skeletal animation

## R-2.9.2 Play Animation Clips

The system SHALL support playing animation clips with loop, clamp, and ping-pong modes. Playback speed and time offset SHALL be configurable per clip.

- **Features:** [F-4.1.4](../../features/4-animation/4.1-animation.md) animation clips

## R-2.9.3 Blend Animation Clips

The system SHALL support blending up to 4 simultaneous animation clips with per-clip weights for crossfade transitions and additive layers.

- **Features:** [F-4.1.4](../../features/4-animation/4.1-animation.md) animation clips, [F-4.1.5](../../features/4-animation/4.1-animation.md) animation state machines

## R-2.9.4 Define Animation State Machines

The system SHALL support defining declarative animation state machines with named states, transition rules, and blend descriptors.

- **Features:** [F-4.1.5](../../features/4-animation/4.1-animation.md) animation state machines

## R-2.9.5 Drive State Machine Parameters

The system SHALL support setting boolean, float, and trigger parameters on animation state machines from game logic. Parameter changes SHALL evaluate transitions within the same frame.

- **Features:** [F-4.1.5](../../features/4-animation/4.1-animation.md) animation state machines

## R-2.9.6 Add Morph Targets

The system SHALL support weighted blend shapes for facial animation, damage deformation, and corrective shapes. Morph target deltas SHALL use half-precision (f16) for position and normal offsets.

- **Features:** [F-4.1.3](../../features/4-animation/4.1-animation.md) morph shapes

## R-2.9.7 Configure IK Chains

The system SHALL support inverse kinematics chains of up to 8 bones with end-effector goals, pole vectors, and GPU FABRIK solver evaluation.

- **Features:** [F-4.1.6](../../features/4-animation/4.1-animation.md) procedural animation

## R-2.9.8 Enable Ragdoll Simulation

The system SHALL support GPU position-based dynamics ragdoll simulation on async compute. CPU readback of ragdoll state SHALL be opt-in.

- **Features:** [F-4.1.6](../../features/4-animation/4.1-animation.md) procedural animation

## R-2.9.9 Animate Instanced Crowds

The system SHALL support batch evaluation of animation for thousands of instances in a single compute dispatch using an arena buffer.

- **Features:** [F-4.1.2](../../features/4-animation/4.1-animation.md) instanced skeletal animation

## R-2.9.10 Define Custom Animation Curves

The system SHALL support user-defined Hermite-interpolated curves with arbitrary float output semantics for controlling material properties, camera parameters, and other runtime values.

- **Features:** [F-4.1.7](../../features/4-animation/4.1-animation.md) custom animation curves

## R-2.9.11 Register Custom Animation Evaluators

The system SHALL support registering user-defined animation evaluator types within the state machine framework. Custom evaluators SHALL receive parameter state and output per-joint transforms or arbitrary float channels.

## R-2.9.12 Enable Cloth Simulation

The system SHALL support GPU position-based dynamics cloth with distance, bending, and self-collision constraints. Cloth SHALL be driven by wind forces, character animation, and collision capsules.

- **Features:** [F-4.1.8](../../features/4-animation/4.1-animation.md) cloth simulation

## R-2.9.13 Enable Hair/Fur Simulation

The system SHALL support physics-based guide strand simulation with wind, gravity, and capsule collision. LOD SHALL transition between full strand simulation at close range and card-based representation at distance.

- **Features:** [F-4.1.9](../../features/4-animation/4.1-animation.md) hair and fur simulation

## R-2.9.14 Enable Motion Matching

The system SHALL support data-driven animation selection that searches a motion capture database for the best continuation of the current pose given a desired trajectory. Motion matching SHALL support character-character and character-environment interactions with GPU-evaluated blending.

- **Features:** [F-4.1.10](../../features/4-animation/4.1-animation.md) motion matching

## R-2.9.15 Enable Muscle Deformation

The system SHALL support secondary muscle deformation driven by joint angles and pose data. Muscle volumes SHALL inflate and deflate to produce anatomically plausible surface deformation layered on top of skeletal skinning.

- **Features:** [F-4.1.11](../../features/4-animation/4.1-animation.md) muscle deformation
