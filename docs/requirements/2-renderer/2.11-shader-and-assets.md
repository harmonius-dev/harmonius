# 2.11 Shader and Assets

## R-2.11.1 Author Shader Graphs

The system SHALL support authoring shader graphs as DAGs of typed nodes connected by typed edges. Graphs SHALL be serializable to a human-readable textual format and support compile-time permutation axes.

- **Features:** [F-6.1.1](../../features/6-tooling-and-io/6.1-shader-and-assets.md) shader graph authoring

## R-2.11.2 Compile Shader Graphs

The system SHALL provide a standalone compiler that transforms shader graphs into platform-native shader code via a platform-neutral intermediate representation. The compiler SHALL support validation, permutation enumeration, reflection data extraction, and cross-compilation.

- **Features:** [F-6.1.2](../../features/6-tooling-and-io/6.1-shader-and-assets.md) shader graph compilation

## R-2.11.3 Register Custom Shader Nodes

The system SHALL support registering third-party shader node types via a typed node registry with slot signatures and IR lowering functions.

- **Features:** [F-6.1.3](../../features/6-tooling-and-io/6.1-shader-and-assets.md) custom shader nodes

## R-2.11.4 Import Source Assets

The system SHALL support importing source assets from Blender, Maya, Photoshop, and Substance via automatic
headless export to interchange formats (glTF 2.0, Alembic, PNG, EXR). The cook pipeline SHALL parse
interchange assets using low-level libraries (cgltf, libalembic, libpng, OpenEXR, libexif), optimize
and meshletize meshes via meshoptimizer, compress geometry with Draco and chunks with zstd, and produce
platform-specific zero-copy binary asset bundles. Asset bundles SHALL be compressed with zstd during transit and
storage. At startup, bundles SHALL be decompressed on the CPU and cached locally so that subsequent loads use
zero-copy memory-mapped reads with no runtime decompression overhead. glTF SHALL be used as an interchange format
for optimized meshes, not as a raw import format.

- **Features:** [F-6.1.5](../../features/6-tooling-and-io/6.1-shader-and-assets.md) asset import pipeline

## R-2.11.5 Readback GPU Data to CPU

The system SHALL support typed asynchronous readback of GPU buffer data to CPU memory via staging buffers and fence synchronization.

- **Features:** [F-6.1.4](../../features/6-tooling-and-io/6.1-shader-and-assets.md) resource CPU readback

## R-2.11.6 Declare Custom Render Passes

The system SHALL support user-declared custom render graph passes specifying resource inputs, outputs, and queue requirements. Custom passes SHALL participate in barrier insertion, resource aliasing, and pass scheduling.

## R-2.11.7 Extend Material Models

The system SHALL support composable material model extensions (clearcoat, subsurface, sheen, anisotropy) implementable via the shader graph system without modifying core lighting shaders.

## R-2.11.8 Validate Shader Graphs

The system SHALL validate shader graphs for type correctness, cycle detection, and resource compatibility before compilation. Invalid graphs SHALL produce typed diagnostic errors.

- **Features:** [F-6.1.2](../../features/6-tooling-and-io/6.1-shader-and-assets.md) shader graph compilation

## R-2.11.9 Author Domain-Specific Shader Language

The system SHALL support a high-level DSL wrapper around the target shading language for modular material and feature composition. The DSL SHALL support composable feature blocks, automatic permutation management, and cross-project shader reuse while compiling to optimal platform-native code.

- **Features:** [F-6.1.6](../../features/6-tooling-and-io/6.1-shader-and-assets.md) domain-specific shader language

## R-2.11.10 Enable Neural Texture Compression

The system SHALL support ML-based texture compression where textures are encoded into neural network weights and decoded per texel at runtime via cooperative vector operations in the fragment shader.

- **Features:** [F-6.1.7](../../features/6-tooling-and-io/6.1-shader-and-assets.md) neural texture compression

## R-2.11.11 Enable Neural Materials

The system SHALL support neural network-based material evaluation where complex multi-layer material stacks are distilled into compact neural representations evaluated via hardware-accelerated cooperative vectors in the fragment shader.

- **Features:** [F-6.1.8](../../features/6-tooling-and-io/6.1-shader-and-assets.md) neural materials
