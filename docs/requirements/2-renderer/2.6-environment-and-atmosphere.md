# 2.6 Environment and Atmosphere

## R-2.6.1 Configure Procedural Sky

The system SHALL support procedural sky rendering driven by sun position, time-of-day, and configurable Rayleigh and Mie scattering parameters.

- **Features:** [F-2.2.1](../../features/2-advanced-rendering/2.2-environment.md) procedural sky

## R-2.6.2 Configure Atmosphere LUT Resolution

The system SHALL allow overriding default resolutions for atmosphere lookup tables: transmittance, multi-scattering, sky-view, and aerial perspective.

- **Features:** [F-2.2.1](../../features/2-advanced-rendering/2.2-environment.md) procedural sky

## R-2.6.3 Enable Volumetric Fog

The system SHALL support frustum-aligned froxel volumetric fog with configurable tile size, depth slice count, and depth distribution exponent.

- **Features:** [F-2.2.2](../../features/2-advanced-rendering/2.2-environment.md) volumetric fog

## R-2.6.4 Place Fog Volumes

The system SHALL support placing local fog volumes with uniform, exponential, or exponential-squared density profiles and configurable height falloff.

- **Features:** [F-2.2.5](../../features/2-advanced-rendering/2.2-environment.md) distance and height fog

## R-2.6.5 Enable Procedural Clouds

The system SHALL support ray-marched volumetric clouds within configurable altitude bounds, driven by 3D noise parameters with temporal reprojection for amortized cost.

- **Features:** [F-2.2.3](../../features/2-advanced-rendering/2.2-environment.md) procedural clouds

## R-2.6.6 Configure FFT Ocean

The system SHALL support FFT-based ocean surface simulation with configurable wind parameters, wave spectrum, foam threshold, and Fresnel blending parameters.

- **Features:** [F-2.2.6](../../features/2-advanced-rendering/2.2-environment.md) water simulation

## R-2.6.7 Select Ocean Reflection Mode

The system SHALL allow selecting the ocean reflection method: SSR, RT, or cubemap probe. Underwater rendering effects SHALL be independently toggleable.

- **Features:** [F-2.2.6](../../features/2-advanced-rendering/2.2-environment.md) water simulation

## R-2.6.8 Configure God Ray Source

The system SHALL allow designating light sources as god ray emitters and selecting between screen-space and volumetric rendering modes.

- **Features:** [F-2.2.4](../../features/2-advanced-rendering/2.2-environment.md) god rays

## R-2.6.9 Set Distance Fog Parameters

The system SHALL support analytical distance fog with configurable start/end range, color, and density curve (linear, exponential, exponential-squared).

- **Features:** [F-2.2.5](../../features/2-advanced-rendering/2.2-environment.md) distance and height fog

## R-2.6.10 Animate Environment Parameters

The system SHALL support keyframing sky, fog, cloud, and ocean parameters over time for dynamic time-of-day and weather transitions.

## R-2.6.11 Import Heterogeneous Volumes

The system SHALL support importing sparse volume data from OpenVDB files. Volumes SHALL be stored as sparse volume textures and ray-marched with emission, absorption, and anisotropic scattering.

- **Features:** [F-2.2.7](../../features/2-advanced-rendering/2.2-environment.md) heterogeneous volumes

## R-2.6.12 Enable Voxel-Based Volumetric Clouds

The system SHALL support full 3D voxel-based volumetric cloud rendering with compressed signed distance fields for ray march acceleration. The system SHALL support ground-level and high-altitude fly-through cloud environments with fluid simulation-based cloud modeling and temporal reprojection over multiple frames.

- **Features:** [F-2.2.8](../../features/2-advanced-rendering/2.2-environment.md) voxel-based volumetric clouds

## R-2.6.13 Configure Breaking Waves

The system SHALL support art-directable breaking wave simulation combining offline-baked deformation textures with runtime compute shader vertex buffer generation. The system SHALL support guide curves for artist control over wave shape and animation.

- **Features:** [F-2.2.9](../../features/2-advanced-rendering/2.2-environment.md) art-directable breaking waves

## R-2.6.14 Configure Weather System

The system SHALL support a dynamic weather state machine driving cloud density, atmospheric scattering, fog, precipitation, wind, and lighting parameters. Weather states SHALL interpolate over configurable transition periods and influence material wetness and vegetation behavior.

- **Features:** [F-2.2.10](../../features/2-advanced-rendering/2.2-environment.md) weather system
