# 2.14 Anti-Aliasing and Upscaling

## R-2.14.1 Configure TAA

The system SHALL support temporal anti-aliasing with per-frame sub-pixel jitter, motion vector consumption, and history clamping to reject disoccluded pixels.

- **Features:** [F-1.5.1](../../features/1-rendering/1.5-anti-aliasing.md) temporal anti-aliasing

## R-2.14.2 Configure Temporal Super Resolution

The system SHALL support rendering at a lower internal resolution and upscaling to display resolution via temporal accumulation with detail reconstruction. Quality/performance scaling SHALL be configurable.

- **Features:** [F-1.5.2](../../features/1-rendering/1.5-anti-aliasing.md) temporal super resolution

## R-2.14.3 Enable FXAA Fallback

The system SHALL support single-pass spatial anti-aliasing via FXAA with no temporal history dependency. FXAA SHALL be available as a fallback when TAA is disabled.

- **Features:** [F-1.5.3](../../features/1-rendering/1.5-anti-aliasing.md) FXAA

## R-2.14.4 Enable Forward-Path MSAA

The system SHALL support 2x and 4x multi-sample anti-aliasing in the forward rendering path only. MSAA SHALL NOT be available in the deferred path.

- **Features:** [F-1.5.4](../../features/1-rendering/1.5-anti-aliasing.md) MSAA

## R-2.14.5 Select Anti-Aliasing Mode

The system SHALL allow selecting the primary anti-aliasing method (TAA, TSR, FXAA, MSAA) globally or per camera. Only one primary AA method SHALL be active per camera at a time.

## R-2.14.6 Enable Checkerboard Rendering

The system SHALL support half-resolution checkerboard rendering with temporal reconstruction to full resolution. The system SHALL render 50% of pixels per frame at alternating positions and reconstruct missing samples from current and previous frame data with a configurable resolve filter.

- **Features:** [F-1.5.5](../../features/1-rendering/1.5-anti-aliasing.md) checkerboard rendering

## R-2.14.7 Integrate Third-Party Upscalers

The system SHALL support integration of vendor-specific temporal upscalers (NVIDIA DLSS, AMD FSR, Intel XeSS) via an abstraction layer. The integration SHALL manage per-vendor initialization, quality mode selection, and fallback to the built-in TSR when no vendor SDK is available.

- **Features:** [F-1.5.6](../../features/1-rendering/1.5-anti-aliasing.md) third-party upscaler integration

## R-2.14.8 Enable Frame Generation

The system SHALL support AI-driven frame interpolation that synthesizes intermediate frames from motion vectors, depth, and history frames. Frame generation SHALL support multi-frame multiplication on capable hardware with latency compensation.

- **Features:** [F-1.5.7](../../features/1-rendering/1.5-anti-aliasing.md) frame generation

## R-2.14.9 Enable Latency Reduction

The system SHALL support end-to-end render pipeline latency optimization that synchronizes CPU submission timing with GPU back-pressure to minimize input-to-display latency.

- **Features:** [F-1.5.8](../../features/1-rendering/1.5-anti-aliasing.md) latency reduction
