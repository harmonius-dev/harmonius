# 2.16 Foliage and Vegetation

## R-2.16.1 Configure Hierarchical Instanced Foliage

The system SHALL support hardware instancing with a hierarchical spatial tree for dense vegetation. Instance data (transform, per-instance random seed, LOD) SHALL be stored in a GPU buffer. The spatial tree SHALL enable efficient frustum and distance culling of millions of instances.

- **Features:** [F-3.3.1](../../features/3-geometry/3.3-foliage-and-vegetation.md) hierarchical instanced rendering

## R-2.16.2 Configure Foliage Wind

The system SHALL support GPU-driven per-instance wind deformation with a global wind field and per-instance phase offsets. Hierarchical wind layers (trunk sway, branch oscillation, leaf flutter) SHALL each have independent frequency and amplitude.

- **Features:** [F-3.3.2](../../features/3-geometry/3.3-foliage-and-vegetation.md) foliage wind animation

## R-2.16.3 Configure Per-Instance Fade

The system SHALL support vertex-shader-computed distance fade with a configurable opacity ramp range before the cull distance threshold, eliminating hard pop-in at LOD and cull boundaries.

- **Features:** [F-3.3.3](../../features/3-geometry/3.3-foliage-and-vegetation.md) per-instance fade

## R-2.16.4 Enable Two-Sided Foliage Transmission

The system SHALL support backlit leaf and petal rendering with a subsurface color parameter for light transmission through thin organic surfaces.

- **Features:** [F-3.3.4](../../features/3-geometry/3.3-foliage-and-vegetation.md) two-sided foliage transmission

## R-2.16.5 Configure Foliage LOD

The system SHALL support per-species LOD chains (full mesh, billboard, culled) with screen-size threshold transitions.

## R-2.16.6 Enable GPU Vegetation Skinning

The system SHALL support full skeleton rigs on vegetation with GPU skinning. Each tree or bush SHALL support a bone hierarchy with spring-based physics for wind-responsive animation. Artists SHALL be able to write custom bone shader code hooking into the GPU skinning API. An SDF-based wind system SHALL provide smooth indoor/outdoor transitions.

- **Features:** [F-3.3.5](../../features/3-geometry/3.3-foliage-and-vegetation.md) GPU vegetation skinning

## R-2.16.7 Enable Interactive Vegetation

The system SHALL support character-vegetation interaction where plants react to character movement and environmental forces. Interaction impulses SHALL propagate through the vegetation skeleton producing persistent displacement trails with configurable decay.

- **Features:** [F-3.3.6](../../features/3-geometry/3.3-foliage-and-vegetation.md) interactive vegetation

## R-2.16.8 Enable Deferred Texturing for Vegetation

The system SHALL support a visibility buffer pre-pass for alpha-tested foliage enabling compute-shader-based material evaluation with software variable rate shading. The pass SHALL overlap with G-Buffer and shadow map rendering on async compute.

- **Features:** [F-3.3.7](../../features/3-geometry/3.3-foliage-and-vegetation.md) deferred texturing for vegetation
