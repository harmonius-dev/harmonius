# 2.4 Materials and Shading

## R-2.4.1 Create PBR Materials

The system SHALL support creating physically-based materials with albedo, normal, metallic, roughness, occlusion, and emissive texture channels accessed via bindless descriptor handles.

- **Features:** [F-1.2.3](../../features/1-rendering/1.2-lighting-and-materials.md) PBR materials

## R-2.4.2 Configure Extended BSDF Layers

The system SHALL support composable BSDF extensions over the base PBR model: subsurface scattering, clearcoat, anisotropy, and sheen. Each layer SHALL be independently enableable per material.

- **Features:** [F-1.2.4](../../features/1-rendering/1.2-lighting-and-materials.md) extended BSDF

## R-2.4.3 Configure SSS Profiles

The system SHALL support per-material subsurface scattering profiles with configurable scatter radius and extinction color. Profiles SHALL drive screen-space or ray-traced SSS evaluation.

- **Features:** [F-1.3.4](../../features/1-rendering/1.3-shadows-and-effects.md) subsurface scattering

## R-2.4.4 Set Alpha Mode

The system SHALL support opaque, alpha-masked (with configurable threshold), and alpha-blended rendering modes per material.

- **Features:** [F-1.3.6](../../features/1-rendering/1.3-shadows-and-effects.md) alpha mask cutouts, [F-1.3.5](../../features/1-rendering/1.3-shadows-and-effects.md) transparent objects

## R-2.4.5 Enable Alpha Mask Cutouts

The system SHALL support alpha-tested cutout rendering where fragments below the alpha threshold are discarded. Cutout geometry SHALL participate in shadow map rendering.

- **Features:** [F-1.3.6](../../features/1-rendering/1.3-shadows-and-effects.md) alpha mask cutouts

## R-2.4.6 Configure Transparency Sorting

The system SHALL automatically sort transparent entities back-to-front before rendering. Users SHALL only need to mark entities as transparent; sorting SHALL be handled by the engine.

- **Features:** [F-1.3.5](../../features/1-rendering/1.3-shadows-and-effects.md) transparent objects

## R-2.4.7 Assign Materials via Shader Graphs

The system SHALL support assigning shader graph instances as materials to entities. Shader graph exposed parameters SHALL be settable per entity.

- **Features:** [F-6.1.1](../../features/6-tooling-and-io/6.1-shader-and-assets.md) shader graph authoring

## R-2.4.8 Override Material Parameters at Runtime

The system SHALL support per-instance material parameter overrides without recompilation. Overrides SHALL be uploaded via a per-instance parameter buffer.

- **Features:** [F-1.2.5](../../features/1-rendering/1.2-lighting-and-materials.md) material instances

## R-2.4.9 Configure LOD Transition Style

The system SHALL support configurable LOD transition methods: dithered crossfade, screen-door transparency, or vertex morphing. Transition range SHALL be configurable per asset.

## R-2.4.10 Create Material Instances

The system SHALL support creating parameterized material instances that override parent material properties (scalars, vectors, textures) without shader recompilation. Instances SHALL share the compiled shader and upload only a per-instance parameter buffer.

- **Features:** [F-1.2.5](../../features/1-rendering/1.2-lighting-and-materials.md) material instances

## R-2.4.11 Configure Material Layers

The system SHALL support multi-layer material compositing where independent material layers are stacked with blend masks. Each layer SHALL be evaluated and composited at the pixel level.

- **Features:** [F-1.2.6](../../features/1-rendering/1.2-lighting-and-materials.md) material layers

## R-2.4.12 Place Decals

The system SHALL support projected box decals writing to the DBuffer or G-buffer before lighting, and mesh decals for proper normal-mapped contact with curved surfaces.

- **Features:** [F-1.2.7](../../features/1-rendering/1.2-lighting-and-materials.md) decal rendering

## R-2.4.13 Select Shading Model Variant

The system SHALL support per-material shading model selection via a material flag. Available models SHALL include: hair (anisotropic Marschner), cloth (fuzz layer with fiber scattering), eye (cornea refraction with iris parallax), thin translucent, two-sided foliage (subsurface transmission), preintegrated skin, and single-layer water.

- **Features:** [F-1.2.8](../../features/1-rendering/1.2-lighting-and-materials.md) shading model variants
