# 7.2 State Tracking

## GR-2.1 Tracked Command Buffer

The GPU runtime SHALL provide a tracked command buffer wrapper that intercepts state-setting
methods and suppresses redundant API calls by comparing against cached state. The tracked
wrapper SHALL forward only state changes that differ from the currently cached state to the
underlying `harmonius::gpu` command buffer.

**Derived from:** R-3.1.2, R-3.1.8

**Rationale:** Redundant state-setting calls (same pipeline bound twice, same viewport set
repeatedly) waste CPU time and may cause unnecessary GPU pipeline flushes.

## GR-2.2 Pipeline State Cache

The tracked command buffer SHALL cache the currently bound pipeline handle for each pipeline
bind point (render, compute, ray tracing) and skip `set_pipeline` calls when the requested
pipeline is already active.

**Derived from:** R-3.1.2

**Rationale:** Pipeline binds are among the most expensive state changes; eliminating redundant
binds reduces CPU and GPU overhead.

## GR-2.3 Descriptor Heap Cache

The tracked command buffer SHALL cache the currently bound descriptor heap handle and skip
`bind_descriptor_heap` calls when the requested heap is already bound.

**Derived from:** R-3.1.2

**Rationale:** Descriptor heap binds invalidate root arguments on D3D12; avoiding unnecessary
rebinds prevents downstream state invalidation.

## GR-2.4 Dynamic State Cache

The tracked command buffer SHALL cache viewport, scissor, and blend constant state. Calls to
`set_viewport`, `set_scissor`, and `set_blend_constants` SHALL be suppressed when the
requested values match the cached state.

**Derived from:** R-3.1.2

**Rationale:** Dynamic state is set per pass but often identical across consecutive passes.

## GR-2.5 Push Constant Cache

The tracked command buffer SHALL cache the most recently written push constant data and skip
`push_constants` calls when the data, offset, and size are identical to the cached values.

**Derived from:** R-3.1.2

**Rationale:** Push constants are set frequently and often contain the same data across
similar draw calls.

## GR-2.6 Resource State Cache

The GPU runtime SHALL maintain a per-resource state cache that tracks the current
layout/access state of every tracked resource. The cache SHALL be used to determine the
minimum set of barriers required before a pass executes, eliding barriers for resources
already in the target state. The cache SHALL be updated after each barrier is emitted.

**Derived from:** R-3.1.2, RG-3.1, RG-3.2

**Rationale:** Emitting barriers for resources already in the correct state wastes GPU
synchronization bandwidth. Tracking state enables optimal barrier insertion.

## GR-2.7 Command Buffer State Reset

The tracked command buffer SHALL reset all cached state to unknown/invalid when `begin()` is
called, ensuring that the first state-setting call in each command buffer is always forwarded.
State SHALL NOT leak across command buffer recording sessions.

**Derived from:** R-3.1.2

**Rationale:** Command buffer state is undefined at the start of recording; assuming any
cached state would cause rendering errors.
