# 3.3 Data Constraints

## R-3.3.1 Reverse-Z Depth

Reverse-Z depth (near=1.0, far=0.0 in NDC) SHALL be used on all platforms. This maximizes floating-point depth precision at distance, which is critical for large open-world scenes.

## R-3.3.2 Skeleton Bone Limit

The maximum number of bones per skeleton SHALL be 256. This covers all production character rigs while fitting within a single structured buffer.

## R-3.3.3 Animation Blend Limit

The maximum number of simultaneously blended animation clips per instance SHALL be 4. This is sufficient for crossfade transitions with additive layers.

## R-3.3.4 IK Chain Length

The maximum IK chain length SHALL be 8 bones, covering standard arm, leg, and spine chains.

## R-3.3.5 Morph Target Precision

Morph target deltas SHALL use half-precision (f16) for position and normal offsets. This provides sub-millimeter accuracy while halving memory and bandwidth versus f32.

## R-3.3.6 Bindless Heap Capacity

The global bindless descriptor heap SHALL support at least 1 million resource slots to accommodate large open-world scenes with many unique assets.

## R-3.3.7 Single-Writer Invariant

During the render phase, each resource SHALL have at most one writer per frame. This eliminates the need for write-write synchronization and simplifies barrier insertion.

## R-3.3.8 Shader IR

A single platform-neutral shader intermediate representation SHALL be used. No hand-authored platform-specific shader code SHALL exist. All shaders SHALL be cross-compiled from this IR to Metal Shading Language, HLSL, and SPIR-V.

## R-3.3.9 Shader Graph Format

Shader graphs SHALL be serialized in a human-readable textual format suitable for version control, diffability, and merge conflict resolution. The format SHALL be deterministic.

## R-3.3.10 Asset Import Format

glTF 2.0 SHALL be the primary 3D asset import format. Standard PBR materials SHALL be automatically converted to shader graphs.

## R-3.3.11 Lighting Model Selection

Forward+ and deferred lighting SHALL be mutually selectable per render graph. Both SHALL NOT be required to run simultaneously.

## R-3.3.12 Animation Pipeline Ordering

Morph target evaluation SHALL precede skeletal skinning in the GPU compute pipeline, following standard glTF/USD conventions.
