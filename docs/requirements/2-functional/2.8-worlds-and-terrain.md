# 2.8 Worlds and Terrain

## R-2.8.1 Configure Streaming World

The system SHALL support chunk-based world streaming with configurable chunk dimensions, LOD levels, and resident pool capacity. Chunks SHALL stream in and out based on camera distance.

- **Features:** [F-3.2.1](../../features/3-geometry/3.2-worlds-and-terrain.md) infinite streamable universes

## R-2.8.2 Register World Chunk Sources

The system SHALL accept chunk data from disk files or procedural generators. Chunk sources SHALL be registered via the streaming API and loaded on demand.

- **Features:** [F-3.2.1](../../features/3-geometry/3.2-worlds-and-terrain.md) infinite streamable universes

## R-2.8.3 Configure Voxel World

The system SHALL support sparse voxel octree worlds with configurable page size, octree depth, and streaming pool capacity.

- **Features:** [F-3.2.2](../../features/3-geometry/3.2-worlds-and-terrain.md) planetary-scale voxel worlds

## R-2.8.4 Modify Voxels at Runtime

The system SHALL support reading and writing voxel data at arbitrary coordinates. Modified regions SHALL automatically trigger re-extraction of isosurface geometry.

- **Features:** [F-3.2.2](../../features/3-geometry/3.2-worlds-and-terrain.md) planetary-scale voxel worlds

## R-2.8.5 Configure Heightmap Terrain

The system SHALL support heightmap terrain with configurable tile size, height scale, and virtual texture page resolution for surface materials.

- **Features:** [F-3.2.3](../../features/3-geometry/3.2-worlds-and-terrain.md) detailed open-world terrains, [F-6.2.2](../../features/6-tooling-and-io/6.2-io-and-streaming.md) terrain tile streaming

## R-2.8.6 Stream Terrain Tiles

The system SHALL stream terrain height tiles and virtual texture pages via a page table with fault-driven loading and LRU eviction.

- **Features:** [F-6.2.2](../../features/6-tooling-and-io/6.2-io-and-streaming.md) terrain tile streaming

## R-2.8.7 Configure Virtual Texture Pages

The system SHALL support virtual texture pages of 128x128 or 256x256 texels (configurable per texture). The page table indirection texture SHALL support at least 16K x 16K logical resolution.

- **Features:** [F-3.2.3](../../features/3-geometry/3.2-worlds-and-terrain.md) detailed open-world terrains

## R-2.8.8 Monitor Streaming Residency

The system SHALL expose per-chunk, per-tile, and per-page streaming residency status. Pool utilization metrics SHALL be queryable for performance monitoring.

## R-2.8.9 Configure HLOD Generation

The system SHALL support automatic proxy mesh generation from static mesh clusters. HLOD proxies SHALL be grid-cell-aligned for streaming world compatibility.

- **Features:** [F-3.2.5](../../features/3-geometry/3.2-worlds-and-terrain.md) hierarchical level of detail

## R-2.8.10 Configure Landscape Material Layers

The system SHALL support multi-layer terrain material blending with per-layer weight maps. Layers SHALL be composited at the pixel level using height-weighted transitions.

- **Features:** [F-3.2.6](../../features/3-geometry/3.2-worlds-and-terrain.md) landscape material layers

## R-2.8.11 Configure Virtual Heightfield Mesh

The system SHALL support runtime tessellation from virtual texture height data for landscape overlays. The mesh SHALL subdivide based on virtual texture resolution and camera distance.

- **Features:** [F-3.2.7](../../features/3-geometry/3.2-worlds-and-terrain.md) virtual heightfield mesh
