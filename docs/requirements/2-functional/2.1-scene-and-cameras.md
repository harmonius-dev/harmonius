# 2.1 Scene and Cameras

## R-2.1.1 Add Entities to Scene

The system SHALL provide a declarative scene API for adding renderable entities. Each entity SHALL be associated with a mesh resource handle, a material, and a transform. Entities added in a given frame SHALL be visible in the next rendered frame.

- **Features:** [F-1.1.2](../../features/1-rendering/1.1-core-rendering.md) GPU frustum culling, [F-1.1.3](../../features/1-rendering/1.1-core-rendering.md) backface culling

## R-2.1.2 Configure Perspective Camera

The system SHALL support perspective camera configuration with field of view, near plane distance, aspect ratio, and optional infinite far plane. Reverse-Z projection SHALL be applied per R-1.3.2.

- **Features:** [F-1.1.6](../../features/1-rendering/1.1-core-rendering.md) perspective projection

## R-2.1.3 Configure Orthographic Camera

The system SHALL support orthographic camera configuration with left, right, top, bottom, near, and far bounds. Orthographic cameras SHALL be usable for 2D rendering, shadow map generation, and top-down views.

- **Features:** [F-1.1.5](../../features/1-rendering/1.1-core-rendering.md) orthographic projection

## R-2.1.4 Set Entity Transforms

The system SHALL allow setting position, rotation, and scale on any entity. Transform changes SHALL be reflected in the next rendered frame. Transforms SHALL be uploaded to the GPU via a per-frame staging buffer.

- **Features:** [F-1.1.7](../../features/1-rendering/1.1-core-rendering.md) GPU-driven instancing

## R-2.1.5 Enable Multi-Camera Rendering

The system SHALL support rendering from multiple cameras per frame, each targeting a separate render target. Cameras SHALL share scene resources (geometry, lights, GI probes) while maintaining independent view/projection state.

- **Features:** [F-6.2.7](../../features/6-tooling-and-io/6.2-io-and-streaming.md) split-screen and multi-window

## R-2.1.6 Configure Render-to-Texture

The system SHALL allow directing any camera's output to an off-screen texture handle. The resulting texture SHALL be usable as a shader resource in subsequent passes within the same frame.

- **Features:** [F-1.1.8](../../features/1-rendering/1.1-core-rendering.md) render-to-texture

## R-2.1.7 Configure Cubemap Captures

The system SHALL allow placing cubemap capture probes at specified world positions. Each probe SHALL render six faces into a cubemap texture. Face update frequency SHALL be configurable (all faces per frame, round-robin, or on-demand).

- **Features:** [F-1.1.9](../../features/1-rendering/1.1-core-rendering.md) cubemaps

## R-2.1.8 Select Lighting Model

The system SHALL allow selecting between forward+ and deferred lighting models per render graph via a configuration enum. The selected model SHALL determine G-buffer allocation and light accumulation strategy.

- **Features:** [F-1.2.1](../../features/1-rendering/1.2-lighting-and-materials.md) forward+ lighting, [F-1.2.2](../../features/1-rendering/1.2-lighting-and-materials.md) deferred lighting

## R-2.1.9 Configure GPU Culling

The system SHALL allow toggling frustum culling, backface culling, and occlusion culling independently per camera. Culling results SHALL be observable via diagnostic queries.

- **Features:** [F-1.1.2](../../features/1-rendering/1.1-core-rendering.md) frustum culling, [F-1.1.3](../../features/1-rendering/1.1-core-rendering.md) backface culling, [F-1.1.4](../../features/1-rendering/1.1-core-rendering.md) occlusion culling

## R-2.1.10 Configure Scene Capture

The system SHALL support placing 2D planar and omnidirectional capture probes that render the scene from arbitrary viewpoints into texture targets. Capture results SHALL be usable as material inputs for mirrors, portals, and minimaps.

- **Features:** [F-1.1.14](../../features/1-rendering/1.1-core-rendering.md) scene capture

## R-2.1.11 Configure Dynamic Resolution

The system SHALL support dynamic resolution scaling driven by GPU timing feedback. Users SHALL configure a target frame budget in milliseconds and min/max screen percentage bounds. The system SHALL automatically adjust internal render resolution within these bounds.

- **Features:** [F-1.1.15](../../features/1-rendering/1.1-core-rendering.md) dynamic resolution

## R-2.1.12 Configure Post-Processing Stack

The system SHALL allow enabling, disabling, and reordering post-processing passes per camera. Each pass SHALL be independently configurable. The pass chain SHALL be declared as part of the render graph.
