# 6.8 Resource Aliasing

## RG-8.1 Lifetime Interval Computation

The render graph compiler SHALL compute the first-write and last-read execution step for each transient resource within a frame. These intervals SHALL be the basis for all aliasing decisions.

**Derived from:** R-2.1.8, R-2.1.12, R-2.7.10, R-2.7.11, R-2.13.1, R-2.13.2

**Rationale:** Aliasing requires precise knowledge of when each resource is live.

## RG-8.2 Aliased Heap Allocation

The render graph SHALL allow two transient resources whose lifetime intervals do not overlap to share the same backing heap range. The compiler SHALL compute a valid aliasing assignment that minimizes peak memory usage.

**Derived from:** R-2.1.6, R-2.1.8, R-2.7.10, R-2.7.11, R-2.12.2, R-2.13.1, R-2.13.2

**Rationale:** Transient render targets, intermediate buffers, and post-process textures can share memory to reduce VRAM usage.

## RG-8.3 Pool-Based Aliasing Domain

Fixed-capacity resource pools SHALL function as aliasing domains. The compiler SHALL assign pool elements with disjoint lifetimes to the same backing heap slot within the pool's reserved capacity.

**Derived from:** R-2.12.2

**Rationale:** Streaming pools with many short-lived elements benefit from intra-pool aliasing.

## RG-8.4 Staging Buffer Ring Aliasing

Staging buffer resources SHALL be aliased within a pre-allocated ring buffer. The ring allocator SHALL partition the buffer into per-frame slots and recycle each slot after the corresponding frame's GPU work completes. Per-frame recycling is sufficient for a triple-buffered pipeline.

**Derived from:** R-2.11.5, R-2.12.3, R-2.12.4, R-2.12.5

**Rationale:** Staging memory is heavily reused each frame; ring aliasing prevents allocation without waste.

## RG-8.5 Heap Type Selection

The render graph SHALL select the appropriate memory heap type (device-local, host-visible, host-cached) for each resource based on its declared usage flags. Aliasing SHALL only occur between resources assigned to the same heap type.

**Derived from:** R-2.11.5, R-2.12.3

**Rationale:** GPU-only and CPU-readable resources occupy different memory types that cannot alias.

## RG-8.6 Memory Usage Diagnostics

The render graph SHALL report per-frame peak aliased memory usage, total allocated memory, and aliasing efficiency (ratio of logical allocation to physical allocation) via the diagnostics API.

**Derived from:** R-2.12.8

**Rationale:** Memory optimization requires visibility into aliasing effectiveness.
