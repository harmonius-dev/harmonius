# 6.9 Multi-View Execution

## RG-9.1 Parameterized Sub-Graph Templates

The render graph SHALL support declaring a sub-graph as a reusable parameterized template. At frame-build time, the template SHALL be instantiated N times, with each instance receiving its own exclusive resource bindings while referencing the same handles for shared resource bindings.

**Derived from:** R-2.1.5, R-2.1.7, R-2.1.10, R-2.2.8, R-2.2.13, R-2.2.15, R-2.3.1, R-2.12.7, R-2.15.7

**Rationale:** Multi-camera, cubemap, cascade, split-screen, and per-light deep opacity map patterns all require repeated execution of identical pass topology with different per-view bindings.

## RG-9.2 Per-Instance Exclusive Resource Binding

Each sub-graph instance SHALL receive its own allocation for resources declared as exclusive. Updating one instance's bindings SHALL NOT require touching other instances' binding tables.

**Derived from:** R-2.1.5, R-2.1.10, R-2.12.7

**Rationale:** Each view needs its own render targets while sharing scene geometry and lighting data.

## RG-9.3 Shared Read-Only Resources Across Instances

Resources declared as shared inputs on a sub-graph template SHALL resolve to a single physical allocation visible to all instances. The graph compiler SHALL emit a single barrier covering all instance reads rather than one barrier per instance, and SHALL verify at most one instance writes a shared resource per frame.

**Derived from:** R-2.1.5, R-2.1.7, R-2.1.10, R-2.2.15, R-2.3.1, R-2.12.7

**Rationale:** Scene geometry, shadow atlases, and GI probes are shared across views; duplicating them or their barriers wastes resources.

## RG-9.4 Array-Layer Instance Targeting

The render graph SHALL support instantiating a sub-graph template N times where each instance targets a distinct layer index of a shared texture array output. The compiler SHALL verify that the array layer count matches the instantiation count.

**Derived from:** R-2.1.7, R-2.3.1

**Rationale:** Cubemap face rendering and cascaded shadow maps populate N layers of one array texture.

## RG-9.5 Multi-Instance Sub-Graph Compilation

The render graph compiler SHALL accept N instantiations of a sub-graph template and compile the combined instance set as a single DAG, merging shared resource nodes, deduplicating barrier edges on shared resources, and producing one topologically sorted execution plan covering all instances.

**Derived from:** R-2.12.7

**Rationale:** Compiling instances as a single DAG enables global optimization of barriers and scheduling across all views.
