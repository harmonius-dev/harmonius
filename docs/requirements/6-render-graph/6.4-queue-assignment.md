# 6.4 Queue Assignment

## RG-4.1 Per-Pass Queue Affinity Declaration

Each pass declaration SHALL accept a queue affinity specifier drawn from the set {graphics, async-compute, transfer, any}. The graph compiler SHALL reject a pass whose declared operations are incompatible with its assigned queue's capability set. Passes with affinity `any` SHALL default to the graphics queue.

**Derived from:** R-2.1.4, R-2.1.9, R-2.2.5, R-2.5.2, R-2.7.5, R-2.11.6, R-2.12.5, R-2.12.6

**Rationale:** Queue assignment determines which hardware engine executes a pass and drives cross-queue synchronization decisions.

## RG-4.2 Dedicated Async Compute Queue

The render graph SHALL expose a named async-compute queue slot whose passes are submitted to a hardware compute queue independent of the graphics queue, enabling GPU-side overlap between compute and graphics command streams within the same frame.

**Derived from:** R-2.2.5, R-2.6.6, R-2.7.5, R-2.8.9, R-2.9.8, R-2.9.12, R-2.9.13, R-2.12.5, R-2.13.5, R-2.17.1, R-2.17.6

**Rationale:** Simulation, culling, histogram, FFT ocean, particle simulation, fluid simulation, and HLOD generation passes benefit from running concurrently with graphics work.

## RG-4.3 Dedicated Transfer Queue

The render graph SHALL expose one or more named transfer queue slots. Transfer passes SHALL operate concurrently with graphics and compute queue passes when the hardware exposes independent transfer engines.

**Derived from:** R-2.1.4, R-2.7.5, R-2.11.4, R-2.12.3, R-2.12.4, R-2.12.6

**Rationale:** Streaming uploads and staging copies must not stall the graphics pipeline.

## RG-4.4 Cross-Queue Dependency Declaration

The render graph SHALL support declaring explicit dependency edges between passes on different queues. The compiler SHALL translate such edges into the appropriate timeline fence signal/wait operations.

**Derived from:** R-2.1.4, R-2.2.5, R-2.6.6, R-2.7.5, R-2.11.4, R-2.12.5, R-2.12.6, R-2.16.1, R-2.17.1

**Rationale:** Cross-queue data flow requires explicit synchronization points expressible in the graph.

## RG-4.5 Queue Capability Fallback

When a pass declares affinity for an async-compute or transfer queue and the hardware does not expose a dedicated queue of that type, the pass SHALL automatically fall back to the graphics queue without requiring graph topology changes.

**Derived from:** R-2.12.5, R-2.12.6

**Rationale:** Not all hardware provides dedicated compute or transfer engines; transparent fallback ensures portability.

## RG-4.6 Queue-Specific Command Buffer Allocation

Each queue type SHALL maintain its own pool of command buffers. Encoding threads SHALL acquire command buffers from the appropriate pool independently, without cross-queue locking.

**Derived from:** R-2.12.3, R-2.12.5, R-2.12.6

**Rationale:** Per-queue pools eliminate contention between graphics, compute, and transfer encoding threads.
