# 6.12 Diagnostics

## RG-12.1 Per-Pass GPU Timestamp Queries

The render graph SHALL support attaching named GPU timestamp queries to any pass. Timestamp pairs (begin, end) SHALL be written to a pre-allocated query pool. Results SHALL be readable by the CPU after the frame fence signals.

**Derived from:** R-2.1.11, R-2.7.7, R-2.12.8

**Rationale:** GPU timing feedback is required for dynamic resolution scaling, performance profiling, and budget gating.

## RG-12.2 Per-Pass Pipeline Statistics Queries

The render graph SHALL support attaching named pipeline statistics query objects to individual passes. Each query SHALL accumulate one or more 64-bit counter values (e.g., primitive counts, invocation counts) readable on the CPU after the frame fence signals.

**Derived from:** R-2.1.9, R-2.2.9, R-2.3.8, R-2.7.7, R-2.7.8

**Rationale:** Culling effectiveness, dispatch counts, light counts, and batching statistics require per-pass GPU counters.

## RG-12.3 Transfer Throughput Statistics

The render graph SHALL accumulate per-transfer-pass per-frame byte counts and GPU-measured latency. These SHALL be aggregated into per-pool and per-queue counters accessible via a statistics query API.

**Derived from:** R-2.8.8, R-2.12.8

**Rationale:** IO throughput monitoring and streaming residency bandwidth tracking require per-pass transfer metrics.

## RG-12.4 Queue Depth Counter

The render graph SHALL expose per-queue integer counters representing the number of passes currently in-flight (submitted but not signaled). Counters SHALL be readable from the host without GPU synchronization.

**Derived from:** R-2.12.8

**Rationale:** Queue saturation monitoring is required for streaming budget decisions.

## RG-12.5 GPU Readback Pass

The render graph SHALL support declaring a pass type that copies a GPU buffer region to a CPU-readable staging buffer and signals a fence on completion. The caller SHALL map and read the staging buffer after the fence resolves.

**Derived from:** R-2.1.9, R-2.1.11, R-2.2.9, R-2.9.8, R-2.11.5

**Rationale:** Diagnostic counters, GPU timing feedback, light count queries, and debug data all require GPU-to-CPU readback.

## RG-12.6 Conditional Debug Overlay Passes

The render graph SHALL support declaring debug overlay passes that are toggled at runtime. When disabled, the pass and all its exclusively-produced resources SHALL be eliminated from the execution plan.

**Derived from:** R-2.7.6

**Rationale:** Debug visualizations must be free when disabled and instantly activatable.

## RG-12.7 Zero-Overhead Diagnostic Opt-Out

All timestamp queries, statistics queries, and debug passes SHALL be individually opt-out at graph compile time. When disabled, the compiler SHALL emit no query commands and SHALL not allocate query pool entries or command buffer space for those passes.

**Derived from:** R-2.7.7, R-2.7.8, R-2.12.8

**Rationale:** Shipping builds must incur zero overhead from diagnostic instrumentation.
