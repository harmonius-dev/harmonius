# 6.7 Budget Culling

## RG-7.1 GPU Timing Feedback Gate

The render graph SHALL support a dynamic gate whose open/closed state is determined by a GPU timestamp delta sampled from a prior frame. A pass annotated with such a gate SHALL be culled from the current frame's execution plan when the measured delta exceeds a configurable threshold.

**Derived from:** R-2.1.11, R-2.3.9

**Rationale:** Dynamic resolution and quality scaling require per-frame culling decisions based on actual GPU performance.

## RG-7.2 Per-Pass Cost and Priority Annotations

Each pass SHALL accept optional cost (estimated GPU time) and priority (culling order) annotations. The budget culling system SHALL use these annotations to determine which passes to cull first under frame-time pressure.

**Derived from:** R-2.12.1, R-2.12.2

**Rationale:** Intelligent culling requires knowing relative pass importance and expected cost.

## RG-7.3 Cascading Dead-Pass Elimination

When a pass is culled by a budget gate, capability gate, or configuration variant, the graph compiler SHALL transitively cull any downstream pass whose only source for a required resource was the culled pass. Resources exclusively produced for culled passes SHALL not be allocated.

**Derived from:** R-2.1.9, R-2.1.12, R-2.3.3, R-2.3.5, R-2.6.5, R-2.7.6, R-2.12.1, R-2.12.2, R-2.12.3, R-2.15.7, R-2.17.1, R-2.17.6

**Rationale:** Culling a producer without culling its exclusive consumers wastes work; transitive elimination prevents this.

## RG-7.4 Resolution Scale Parameter

The render graph SHALL maintain a per-frame scalar resolution-scale parameter in the range [min, max] applied to all resources declared with resolution-scaled dimensions. An external controller SHALL update this parameter each frame without triggering graph recompilation.

**Derived from:** R-2.1.11

**Rationale:** Dynamic resolution scaling adjusts render target sizes to meet frame-time budgets.

## RG-7.5 Per-Pool Utilization Budget Gate

Each fixed-capacity resource pool SHALL expose a utilization ratio as a dynamic gate input. Passes may declare a budget gate evaluating pool utilization against a threshold; passes failing this gate SHALL be culled along with their exclusive dependents.

**Derived from:** R-2.12.2

**Rationale:** Pool pressure must be manageable by culling low-priority work that would exceed pool capacity.

## RG-7.6 Runtime Parameter Mutation Without Recompilation

The render graph SHALL support updating dynamic gate parameters, resolution scale, and pass enable flags per frame without triggering graph recompilation. Only the execution plan's active pass set and resource dimensions SHALL be affected.

**Derived from:** R-2.1.11, R-2.1.12, R-2.7.6

**Rationale:** Frame-to-frame quality adjustments must be instantaneous without recompilation overhead.
