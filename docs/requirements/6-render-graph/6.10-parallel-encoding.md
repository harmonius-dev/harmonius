# 6.10 Parallel Encoding

## RG-10.1 Independent Command Buffers Per Pass

The render graph executor SHALL allocate an independent command buffer for each pass or pass group assigned to the same queue segment. No two concurrently-encoding passes SHALL write to a shared command buffer.

**Derived from:** R-2.1.5, R-2.1.7, R-2.12.7

**Rationale:** Parallel encoding requires each recording thread to operate on its own command buffer.

## RG-10.2 Thread-Safe Command Buffer Pool

The render graph SHALL maintain per-queue-type thread-safe pools of command buffers. Encoding threads SHALL acquire command buffers from the appropriate pool independently without cross-queue or cross-thread locking.

**Derived from:** R-2.12.3, R-2.12.4, R-2.12.5, R-2.12.6

**Rationale:** Lock-free pool access is required for parallel encoding to scale.

## RG-10.3 Sub-Graph Instance Parallel Encoding

Passes belonging to independent sub-graph instances that share no write-write or write-read dependency SHALL be eligible for concurrent encoding on separate threads. The executor SHALL schedule such passes to separate encoding threads when ready simultaneously.

**Derived from:** R-2.1.5, R-2.1.7, R-2.3.1, R-2.12.7

**Rationale:** Multi-camera, cubemap, and cascaded shadow rendering should encode views in parallel.

## RG-10.4 Encoding Dependency Graph

The render graph SHALL compute an encoding dependency graph that identifies which passes can be encoded in parallel and which must be serialized. The dependency graph SHALL consider both resource dependencies and queue submission ordering.

**Derived from:** R-2.7.5, R-2.12.7

**Rationale:** Optimal thread utilization requires knowing the parallelism available in the pass graph.

## RG-10.5 Per-Frame Ring Buffer Allocation

All per-frame transient data (constants, staging regions) SHALL be allocated from pre-allocated ring buffers partitioned by frame index. Ring buffer allocation SHALL be lock-free and zero-allocation on the per-frame hot path.

**Derived from:** R-2.1.4, R-2.12.3, R-2.12.4, R-2.12.5

**Rationale:** Zero-allocation execution requires pre-allocated ring-buffered memory with frame-indexed recycling.

## RG-10.6 Timeline Fence Coordination

The render graph SHALL use timeline fence primitives (monotonically increasing signal values) to coordinate completion of passes on one queue with the start of dependent passes on another queue. Each frame SHALL maintain per-queue timeline fence counters advanced at submission time.

**Derived from:** R-2.7.5, R-2.12.5, R-2.12.6

**Rationale:** Cross-queue synchronization requires timeline fences; per-queue counters enable fine-grained signaling.

## RG-10.7 Submission Ordering Separate from Encoding Order

The render graph SHALL support submitting command buffers to the GPU in topological order independently of the order in which they were encoded. Encoding order SHALL be determined by thread scheduling; submission order SHALL be determined by the compiled execution plan.

**Derived from:** R-2.1.5, R-2.12.7

**Rationale:** Parallel encoding produces command buffers out of topological order; submission must reorder them.
